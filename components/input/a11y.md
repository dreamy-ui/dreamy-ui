# Input — Accessibility Requirements

Single-line text field. Dreamy renders a native `<input>` (plus optional `Input.Group` addons). Prefer native semantics over ARIA roles.

**APG:** Prefer native HTML text inputs; see [APG Practices / form patterns overview](https://www.w3.org/WAI/ARIA/apg/patterns/) and HTML text field guidance. No custom `role` required on a native input.

---

## Dreamy form composition

Standalone control; compose **inside `Field`**.

| Concern | Owned by | Not owned by Input |
| --- | --- | --- |
| Visible label | `Field.Label` | Do not add a built-in Field label |
| Hint / error | `Field.Hint` / `Field.Error` | Do not bake into Input |
| `id`, `aria-describedby`, invalid, required | Field → Input via `useField` | Input consumes Field |

```tsx
<Field>
  <Field.Label>Email</Field.Label>
  <Input type="email" autoComplete="email" />
  <Field.Hint>We’ll never share it.</Field.Hint>
  <Field.Error>Enter a valid email.</Field.Error>
</Field>
```

Placeholder is **not** a label. The control must expose name/role/value and Field wiring.

## Anatomy

| Part | Element | A11y role |
| --- | --- | --- |
| **Input** | native `<input>` | Textbox (implicit) — focus + value |
| **Group** | `Input.Group` | Layout container |
| **Prefix / Suffix** | decorative slots | Usually presentational; hide icons with `aria-hidden` |
| **StartAddon / EndAddon** | adjacent chrome | Must not steal the input’s name; keep non-interactive or properly labeled if interactive |

## Requirements

### Role & value

- Native `<input>` — correct `type` (`text`, `email`, `password`, `search`, `tel`, `url`, `number`, …).
- Current value via the value property; AT reads it as the textbox value.
- Do not add `role="textbox"` unless replacing a non-input element (not Dreamy’s model).

### Naming & descriptions

- Accessible name from `Field.Label` (`htmlFor`/`id`) or `aria-label` / `aria-labelledby`.
- Hints/errors via `aria-describedby` from Field.
- `aria-invalid` when Field `isInvalid`.
- `required` / `aria-required` from Field.
- `disabled` / `readOnly` as appropriate.

### Keyboard

| Key | Behavior |
| --- | --- |
| **Tab** | Focus input |
| Printable keys | Enter text (per type) |
| **Arrow / Home / End** | Caret movement (native) |

No keyboard trap. Group addons that are buttons need their own names and tab stops.

### Autocomplete & purpose

Where collecting personal data, set meaningful `autoComplete` tokens (WCAG 1.3.5 Identify Input Purpose).

## Checklist

- [ ] Native input is the focusable control
- [ ] Field.Label (or aria-label) provides accessible name
- [ ] Placeholder not used as the only label
- [ ] `aria-describedby` includes hint and error when present
- [ ] `aria-invalid` / required / disabled wired from Field
- [ ] Correct `type` and `autoComplete` when applicable
- [ ] Prefix/suffix icons are `aria-hidden` if decorative
- [ ] Interactive addons are labeled buttons
- [ ] Visible focus indicator
