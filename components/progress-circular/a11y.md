# Progress Circular — Accessibility Requirements

Guide for implementing and testing accessibility of the Dreamy UI `ProgressCircular` component.

**APG note:** Circular progress uses the same semantics as a linear bar — [`role="progressbar"`](https://www.w3.org/TR/wai-aria-1.2/#progressbar) with valuemin/max/now. Do **not** use the [Meter pattern](https://www.w3.org/WAI/ARIA/apg/patterns/meter/) for loading/task progress.

---

## 1. Purpose and semantics

A circular progress indicator shows completion within a range (or indeterminate activity). Assistive tech must receive **name**, **role=`progressbar`**, and **value** — the SVG rings are visual only.

---

## 2. Component anatomy

| Part | Element (Dreamy) | Role in a11y tree | Purpose |
| --- | --- | --- | --- |
| **Root** | `<div>` via `getProgressBarProps` | **Should be `progressbar`** | Owns name + value |
| **SVG** | `<svg aria-hidden>` | Hidden | Visual track + indicator |
| **Track / Indicator** | `<circle>` | Presentation | Decorative strokes |
| **Value text** | `<span>` (`showValueLabel`) | Text | Visible percent / custom label |
| **Label** | `<span>` (`label`) | Text / labelling | Visible process name |

```
div[role=progressbar][aria-valuemin|max|now][aria-label|labelledby]
  → svg[aria-hidden] → circles
  → optional valueText
  → optional label
```

---

## 3. Requirements

### Role and values

Expose on the **root** (not on SVG circles):

| Attribute | Determinate | Indeterminate |
| --- | --- | --- |
| `role` | `progressbar` | `progressbar` |
| `aria-valuemin` | `minValue` (default `0`) | same |
| `aria-valuemax` | `maxValue` (default `100`) | same |
| `aria-valuenow` | current `value` | **omit** |
| `aria-valuetext` | when percent is insufficient (`valueLabel` / formatted text) | optional |

- SVG and circles: `aria-hidden="true"` / `role="presentation"` so AT does not treat graphics as separate images.
- Keep visual arc in sync with `aria-valuenow`.

### Naming

- Provide a name: visible `label` wired with `aria-labelledby`, or `aria-label` on the progressbar.
- If `showValueLabel` shows a percent, ensure `aria-valuenow` / `aria-valuetext` agree with that text (WCAG 2.5.3).
- Name the **process** (“Loading results”), not only “progress”.

### Indeterminate

- When `value` is undefined and indeterminate: omit `aria-valuenow`; animation is decorative.
- Respect `prefers-reduced-motion` for spinning/indeterminate motion.

### Keyboard

- Not a keyboard widget unless made interactive; no required keys.

## 5. Checklist

- [x] Root exposes `role="progressbar"`
- [x] Accessible name (`label` / `aria-label` / `aria-labelledby`)
- [x] `aria-valuemin` / `aria-valuemax` set
- [x] Determinate: `aria-valuenow`; indeterminate: omitted
- [x] `aria-valuetext` / value label consistent when shown
- [x] SVG decorative (`aria-hidden`)
- [x] Visual value matches exposed ARIA value
- [x] Reduced motion respected for indeterminate animation
