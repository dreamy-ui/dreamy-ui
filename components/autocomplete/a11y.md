# Autocomplete — Accessibility Requirements

Editable combobox: text input filters a popup list of suggestions. Built from `Autocomplete.Input` + listbox content.

**APG:** [Combobox Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/combobox/) (editable combobox with listbox popup)

---

## Dreamy form composition

Standalone control; compose **inside `Field`**. Field owns label/hint/error. Do not add Field chrome into Autocomplete.

Wire Field → the **combobox input**: `id`, `aria-describedby`, `aria-invalid`, required/disabled.

```tsx
<Field>
  <Field.Label>City</Field.Label>
  <Autocomplete.Root items={items}>
    <Autocomplete.Input placeholder="Search…" />
    <Autocomplete.Content />
  </Autocomplete.Root>
  <Field.Error>Select a city.</Field.Error>
</Field>
```

## Anatomy

| Part | Component | A11y role |
| --- | --- | --- |
| **Root** | `Autocomplete.Root` | Container |
| **Control** | input wrapper | Layout |
| **Input** | `Autocomplete.Input` (native input) | **`combobox`** — focus + typed value |
| **Content** | `Content` / `VirtualContent` | **`listbox`** |
| **Item** | `Autocomplete.Item` | **`option`** |
| **Clear** | clear button | Named button |
| **Indicators / icons** | chevron, check, leading icon | Decorative or separately labeled |

### Connections

```
Field.Label ──► combobox input (id)
Field.Hint+Error ──► input (aria-describedby)
input aria-expanded / aria-controls / aria-activedescendant ──► listbox + focused option
option aria-selected ──► current selection
```

## Requirements

### Roles & state

| Element | Requirement |
| --- | --- |
| Input | `role="combobox"`, `aria-expanded`, `aria-haspopup="listbox"` |
| Input | `aria-controls` → listbox id; `aria-activedescendant` → focused option id when navigating |
| Input | `autocomplete="off"` (or appropriate) to avoid browser UI fighting the widget |
| Listbox | `role="listbox"` |
| Options | `role="option"`, `aria-selected` |
| Empty state | “No results” text available to AT when list is empty |

While open and filtering, the input shows the search string; when closed, it may show the selected label — ensure AT still understands selection vs query (combobox value + selected option).

### Naming

- Field.Label names the combobox.
- Options named by their label content.
- Clear: accessible name (e.g. “Clear selection”).
- Leading icon: `aria-hidden` if decorative.

### Keyboard

| Key | Behavior |
| --- | --- |
| **Tab** | Focus input; closes list on leave |
| **ArrowDown** / **ArrowUp** | Open if needed; move active option |
| **Enter** | Select active option; close |
| **Escape** | Close list; keep focus on input |
| Printable keys | Filter; open list as needed |

Focus stays on the input while navigating options (activedescendant model). Do not trap Tab inside the popup. On option click, prevent input blur from swallowing selection (mousedown preventDefault on list is acceptable).

### Clear & disabled

Clear must not remove Field labeling. Disabled/read-only: no open/select. Invalid: `aria-invalid` on the combobox input.

## Checklist

- [ ] Input is `role="combobox"` with expanded/haspopup
- [ ] Popup is listbox; items are options with selected state
- [ ] `aria-controls` / `aria-activedescendant` (or equivalent) wired when navigating
- [ ] Field label/describedby/invalid on the input
- [ ] Arrow / Enter / Escape / Tab match APG editable combobox
- [ ] Focus remains on input during option navigation
- [ ] Empty results announced or otherwise available to AT
- [ ] Clear button named; decorative icons hidden
- [ ] Virtualized list scrolls active option into view
