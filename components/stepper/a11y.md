# Stepper — Accessibility Requirements

Guide for implementing and testing accessibility of the Dreamy UI `Stepper` component.

**Related APG:** no dedicated stepper pattern; align with list semantics, [`aria-current`](https://www.w3.org/TR/wai-aria-1.2/#aria-current), and button/keyboard practices from the [APG patterns index](https://www.w3.org/WAI/ARIA/apg/patterns/).

---

## 1. Purpose and semantics

A stepper shows a multi-step process and which step is **current**, plus completed / incomplete state. Users must hear the current step and operate next/prev (and optional step triggers) by keyboard.

---

## 2. Component anatomy

| Part | Element (Dreamy) | Role in a11y tree | Purpose |
| --- | --- | --- | --- |
| **Root** | `<div>` | Group / container | Process wrapper (`data-orientation`) |
| **List** | `<ol>` | List | Ordered steps |
| **Item** | `<li>` | List item | One step |
| **Trigger** | `<div>` | Step header; `aria-current="step"` when current | Labels / activates step UI |
| **Indicator** | `<div>` | Presentational number/check | Visual status |
| **Title** / **Description** | `<p>` | Text | Step name and help |
| **Separator** | `<div role="separator">` | Separator | Visual connector |
| **Content** | panel (`role="tabpanel"`) | Region for active step body | Shown when `step === index` |
| **NextTrigger** / **PrevTrigger** | `<button>` | Button | Advance / go back |
| **CompletedContent** | `<div>` | Text region | Shown when process complete |

```
Root → ol → li → Trigger[aria-current=step?] + Indicator + Title
              → Content (active only) | Next/Prev buttons
```

---

## 3. Requirements

### Structure and current step

- Steps live in an ordered list so sequence is programmatic.
- Current step trigger exposes `aria-current="step"` (Dreamy `getTriggerProps`).
- Status (complete / current / incomplete) must not rely on color alone; pair with text, icons (`aria-hidden` if decorative), or explicit status text.

### Naming

- Each step needs a clear accessible name (visible `Title`, or `aria-label` on the trigger).
- Next/Prev buttons need names (`aria-label` or visible text). Disable at first/last as appropriate.
- Check icons in indicators: `aria-hidden="true"` so they don’t replace the step name.

### Content panels

- Only the active step content is available (hidden inactive panels).
- If panels use `role="tabpanel"`, wire `aria-labelledby` to the corresponding trigger `id` (Dreamy `getContentProps`).
- Prefer one coherent pattern: either wizard buttons only, or also keyboard-activatable step triggers — document and keep consistent.

### Keyboard

| Key | Requirement |
| --- | --- |
| **Tab** / **Shift+Tab** | Reach prev/next (and any interactive step triggers) |
| **Enter** / **Space** | Activate focused button / trigger |

- Announce or expose current step when it changes (`aria-current`, updated title, optional polite live region for “Step 2 of 4”).
- Focus management: on step change, move focus into the new panel or keep it on Next/Prev consistently.

---

## 5. Checklist

- [ ] Steps in an ordered list
- [ ] Current step has `aria-current="step"`
- [ ] Each step has an accessible name (Title / label)
- [ ] Status not conveyed by color alone
- [ ] Next/Prev are named buttons; correctly disabled
- [ ] Indicator icons don’t override names
- [ ] Active content associated with its step; inactive content not exposed incorrectly
- [ ] Keyboard operable; focus visible; sensible focus on step change
