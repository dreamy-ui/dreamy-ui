# Divider — Accessibility

`Divider` renders a thematic break as `<hr>`. It is **not** an APG interactive widget. Accessibility is about correct separator semantics and orientation.

---

## Semantics

- Native `<hr>` exposes separator semantics (thematic break) by default.
- Orientation (`horizontal` / `vertical`) is primarily presentational; ensure it still reads as a separator when used between content.
- For non-`hr` separators elsewhere, `role="separator"` is the ARIA equivalent — Dreamy’s `Divider` already uses `<hr>`, so extra `role="separator"` is usually unnecessary.

---

## Authoring checklist

| Do | Don't |
| --- | --- |
| Use between major content sections when a break is meaningful | Overuse between every list item (noise for AT) |
| Prefer decorative alternatives (`aria-hidden`) for purely visual rules | Make the divider focusable or interactive |
| Keep surrounding content order clear without the line | Rely on the divider alone to convey hierarchy |

If a decorative line must not be announced, use a non-`hr` element with `aria-hidden="true"` instead of `Divider`, or hide it intentionally.

---

## WCAG notes

| Criterion | Relevance |
| --- | --- |
| **1.3.1** Info and Relationships | `<hr>` conveys a thematic break; decorative lines should not |
| **1.3.2** Meaningful Sequence | Divider does not reorder content; placement should match intent |
| **1.3.3** Sensory Characteristics | Do not instruct users by “the line above” alone |
