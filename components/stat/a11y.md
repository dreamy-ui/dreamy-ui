# Stat — Accessibility Requirements

Guide for implementing and testing accessibility of the Dreamy UI `Stat` compound component.

**Metric display**, not an interactive widget. Semantics come from a description list (`dl` / `dt` / `dd`). Trends and emphasis must not rely on color alone; icons need text alternatives.

There is no dedicated APG “stat” pattern — treat as structured text / [1.3.1 Info and Relationships](https://www.w3.org/WAI/WCAG22/Understanding/info-and-relationships.html). Group related stats with a heading or `aria-labelledby` region when needed ([Landmarks](https://www.w3.org/WAI/ARIA/apg/patterns/) / document structure).

---

## 1. Purpose and semantics

| Part | Dreamy export | Element | Role in a11y tree |
| --- | --- | --- | --- |
| Root | `Stat.Root` | `<dl>` | Groups term + value |
| Label | `Stat.Label` | `<dt>` | Metric name |
| Value | `Stat.ValueText` | `<dd>` | Primary numeric / text value |
| Unit | `Stat.ValueUnit` | `<span>` | Unit adjacent to value (%, USD, …) |
| Hint | `Stat.Hint` | `<span>` | Secondary helper text |
| Up / Down | `Stat.UpIndicator` / `DownIndicator` | `<span>` + icon | Trend — needs a text alternative |

Keep the `dl` → `dt` / `dd` pairing. Do not break the list with unrelated interactive wrappers that split term from definition.

---

## 2. Text alternatives for metrics (required)

| Concern | Requirement |
| --- | --- |
| Label + value | Visible label (`Stat.Label`) and value (`Stat.ValueText`) must be present in text (or equivalent accessible text) |
| Unit | Prefer visible unit text in `ValueUnit`, not color/position alone |
| Trend icons | Arrow icons are non-text content ([1.1.1](https://www.w3.org/WAI/WCAG22/Understanding/non-text-content.html)): provide accessible text such as visually hidden “up” / “down” / “increased” / “decreased”, or `aria-label` on the indicator; decorative-only icons → `aria-hidden="true"` **only if** the same meaning is already in adjacent text |
| Live updates | If values update asynchronously, use a polite live region on the changing text when the update matters ([4.1.3](https://www.w3.org/WAI/WCAG22/Understanding/status-messages.html)) |

---

## 3. Do not rely on color alone (required)

Up/down indicators often use green/red. That fails [1.4.1 Use of Color](https://www.w3.org/WAI/WCAG22/Understanding/use-of-color.html) unless paired with:

- Distinct icons (arrows) **and**
- Text (visible or AT-only) stating direction / change

Shape + text beat color-only meaning ([1.3.3 Sensory Characteristics](https://www.w3.org/WAI/WCAG22/Understanding/sensory-characteristics.html)).

---

## 4. Contrast and non-interactive nature

- Text contrast AA for label, value, hint, unit ([1.4.3](https://www.w3.org/WAI/WCAG22/Understanding/contrast-minimum.html)).
- Indicator icons: non-text contrast ≥ 3:1 ([1.4.11](https://www.w3.org/WAI/WCAG22/Understanding/non-text-contrast.html)).
- Stat parts are **not** focusable controls. If a whole card is clickable, the interactive wrapper (link/button) owns keyboard and naming — not the `dl` internals.

## 5. WCAG 2.2 (relevant)

| SC | Level | Why |
| --- | --- | --- |
| **1.1.1 Non-text Content** | A | Trend icons need alternatives |
| **1.3.1 Info and Relationships** | A | `dl` / `dt` / `dd` structure |
| **1.3.3 Sensory Characteristics** | A | Instructions/meaning not by color/position alone |
| **1.4.1 Use of Color** | A | Trend not by color alone |
| **1.4.3 Contrast (Minimum)** | AA | Label / value / hint text |
| **1.4.11 Non-text Contrast** | AA | Indicator glyphs |
| **4.1.2 Name, Role, Value** | A | Programmatic term/definition; status if live |
| **4.1.3 Status Messages** | AA | When values announce as status updates |

---

## 6. Checklist

- [x] Root is a `dl`; label `dt`; value `dd`
- [x] Metric has visible (or accessible) label and value text
- [x] Units are textual, not implied only by styling
- [x] Up/Down icons have text alternatives or adjacent equivalent text
- [x] Trend meaning does not depend on color alone
- [x] Text and icon contrast meet AA
- [x] No accidental focusable chrome on presentational parts
- [x] Dynamic updates use an appropriate live region when needed
