# Radio Card ‚Äî Accessibility Requirements

Card-styled radio option with title/description and a visual radio indicator. Same semantics as Radio; larger hit target and richer content.

**APG:** [Radio Group Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/radio/)

---

## Dreamy form composition

Standalone control; compose inside **`Field`** + **`RadioGroup`**. Field owns group label/hint/error. `RadioCard.Title` / `Description` are in-card content ‚Äî do not treat them as Field chrome replacements for the whole form field, but they **must** participate in the option‚Äôs accessible name/description.

Wire Field ‚Üí hidden input: `id`, `aria-describedby`, `aria-invalid`, disabled/required.

## Requirements

### Role & state

- Native `input[type="radio"]` is the widget.
- Shared `name` across cards in the group; unique `value` per card.
- Checked via native `checked`. Root uses `data-*` for visuals only.

### Grouping

Must be used in a set with **radiogroup** semantics (see Radio guide): `role="radiogroup"` + labelledby, or fieldset/legend. `RadioGroup` context alone is not enough for AT unless the DOM exposes the group role/name.

### Naming

- Group label names the question/set.
- Title/Label (or `aria-label`) names each option.
- Description linked via `aria-describedby` when meaningful.

### Keyboard

Same as Radio Group APG: **Tab** into group, **arrows** between options, **Space** to check. Focus may paint on Root while DOM focus stays on the input.
