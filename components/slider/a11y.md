## Dreamy form composition

Slider is a **standalone control**. Compose it **inside `Field`**. Do **not** add Field label/hint/error chrome into Slider.

| Concern | Owned by | Slider must |
| --- | --- | --- |
| Visible label | `Field.Label` | Consume Field `id` / `aria-labelledby` on the thumb |
| Hint / error | `Field.Hint` / `Field.Error` | Merge into `aria-describedby` (via Field / hidden input) |
| Required / invalid / disabled | Field flags | Reflect on thumb + hidden input |

```tsx
<Field>
  <Field.Label>Volume</Field.Label>
  <Slider.Root>
    <Slider.Track><Slider.FilledTrack /></Slider.Track>
    <Slider.Thumb />
  </Slider.Root>
  <Field.Hint>0â€“100</Field.Hint>
</Field>
```

## 1. Purpose and semantics

A slider lets the user select a **single numeric value** from a continuous or discrete range.

- One focusable thumb with `role="slider"`.
- Value exposed via `aria-valuenow`, `aria-valuemin`, `aria-valuemax`, and optionally `aria-valuetext`.
- Orientation via `aria-orientation` (`horizontal` | `vertical`).

---

## 2. Component anatomy (multipart)

| Part | Element (Dreamy) | Role in a11y tree | Purpose |
| --- | --- | --- | --- |
| **Root** | `<div>` | Container (`aria-disabled` when disabled) | Layout / orientation |
| **Hidden input** | visually hidden `<input>` | Form value; not the slider widget | Submit value; Field `aria-invalid` |
| **Track** | `<div data-part="track">` | Presentational | Visual range |
| **FilledTrack** | `<div data-part="trackFilled">` | Presentational | Filled portion |
| **Thumb** | `<div data-part="thumb">` | **`role="slider"`** â€” focus + keyboard | Interactive value control |
| **Mark** | optional marker | Presentational (or labeled if interactive) | Tick marks |

**Rules:**

1. **Only the thumb** is the interactive slider in the accessibility tree.
2. Track / filled track / marks are not focusable and must not steal the name/role.
3. Accessible name comes from **Field.Label** (preferred) or `aria-label` / `aria-labelledby` on the thumb.
4. When `hideThumb` is true, ensure an alternative operable focus target still exists (or do not use that mode for keyboard-critical UIs).

---

## 3. Per-part requirements

### Thumb (`role="slider"`)

| Attribute | Requirement |
| --- | --- |
| `aria-valuenow` / `min` / `max` | Always set; stay in sync with value |
| `aria-valuetext` | When the raw number is not meaningful (e.g. â€śMediumâ€ť) |
| `aria-orientation` | Match visual orientation |
| `aria-disabled` / `aria-readonly` | From props / Field |
| `aria-label` / `aria-labelledby` | Name required; prefer Field label |

### Hidden input

- Carries form `name` / `value` and Field `aria-invalid` / `id` wiring.
- Must not create a second competing tab stop for the same control (typically not focusable).

### Track / FilledTrack / Mark

- Decorative relative to the thumb; no conflicting roles.

---

## 4. Keyboard interaction

| Key | Requirement |
| --- | --- |
| **Tab** / **Shift+Tab** | Focus moves to / from the thumb |
| **Arrow keys** | Increase / decrease by step (respect orientation) |
| **Home** / **End** | Jump to min / max |
| **Page Up** / **Page Down** | Larger step (if implemented) |

Also: no keyboard trap; disabled/read-only must not change value; pointer drag must have a keyboard-equivalent path (WCAG 2.5.7).

## 6. Checklist

- [x] Used inside **Field** (no Field chrome inside Slider)
- [x] Thumb has `role="slider"` and correct `aria-value*`
- [x] Accessible name from Field or `aria-label`
- [x] Arrow / Home / End work; Tab moves focus
- [x] Disabled / read-only / invalid reflected
- [x] Focus visible; reduced motion respected for thumb animation
