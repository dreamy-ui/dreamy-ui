# Stack — Accessibility

`Stack`, `HStack`, and `VStack` are layout primitives (`div`) with optional separators. They are **not** APG interactive widgets. A11y is about child semantics, reading order, and separator presentation.

---

## Semantics

- Generic flex/stack containers; no default ARIA role.
- Spacing and direction are visual only — they do not create lists or groups in the accessibility tree.
- Optional `separator` is decorative layout chrome unless you pass a semantic node (e.g. `Divider` / `<hr>`).

---

## Separators

| Separator type | Guidance |
| --- | --- |
| Decorative (icon, plain text, styled span) | Prefer `aria-hidden="true"` so it is not announced between every item |
| Semantic (`Divider` / `<hr>`) | Appropriate when a true thematic break is intended |
| Between related controls | Do not rely on separators alone to group or name controls |

---

## Authoring checklist

| Do | Don't |
| --- | --- |
| Stack semantic children (buttons, fields, text) | Use Stack as a fake list without `<ul>`/`role="list"` when list semantics matter |
| Keep DOM order aligned with reading order | Convey hierarchy only with visual gap/alignment |
| Hide purely decorative separators from AT | Insert focusable separators without a clear purpose |

---

## WCAG notes

| Criterion | Relevance |
| --- | --- |
| **1.3.1** Info and Relationships | Structure from real elements; separators should not invent false relationships |
| **1.3.2** Meaningful Sequence | Stack order in the DOM is the reading order |
| **1.3.3** Sensory Characteristics | Do not instruct users by position alone (“the item on the right”) |
