# Tooltip — Accessibility Requirements

Guide for implementing and testing accessibility of the Dreamy UI `Tooltip` component.

**APG pattern:** [Tooltip](https://www.w3.org/WAI/ARIA/apg/patterns/tooltip/)

---

## 1. Purpose and semantics

A tooltip shows **brief supplemental information** when the trigger is hovered or focused. It is not a dialog and must not contain interactive controls.

- Content: `role="tooltip"`.
- Trigger: `aria-describedby` → tooltip id while open.
- Optional `aria-label` prop: visible `content` may differ from the string exposed to AT (use carefully; visible text should still make sense).

---

## 2. Component anatomy

| Part | Purpose in a11y |
| --- | --- |
| **Root** (internal) | Wrapper |
| **Trigger** | Hover/focus target; may wrap children in a focusable `<span tabIndex={0}>` when `shouldWrapChildren` |
| **Content** | `role="tooltip"` floating label |
| **Arrow** | Decorative |
| **SR-only label** | Used when `aria-label` overrides what AT reads |

---

## 3. Requirements

### Behavior

- Show on **pointer hover** and **keyboard focus**; hide on leave/blur.
- **Escape** hides the tooltip while open (`closeOnEsc`).
- Content is not focusable and has **no** interactive children.
- Do not use `aria-modal` or focus trap.
- Empty `content`: render children only (no empty tooltip).
- Meet WCAG **1.4.13**: dismissible (Escape), hoverable (pointer can move onto tooltip if it would otherwise dismiss), persistent while hover/focus remains.

### Naming

- Prefer associating the visible tooltip text via `aria-describedby`.
- If `aria-label` is used for AT, keep the visible content non-conflicting for sighted users.

### Keyboard

| Key | Requirement |
| --- | --- |
| **Focus** | Shows tooltip |
| **Escape** | Hides tooltip |
| **Blur** | Hides tooltip |

## 4. WCAG (relevant)

| SC | Level | Notes |
| --- | --- | --- |
| **1.4.13 Content on Hover or Focus** | AA | Dismiss / hover / persist |
| **2.1.1 Keyboard** | A | Available on focus |
| **1.4.3 / 1.4.11** | AA | Text and non-text contrast |
| **4.1.2 Name, Role, Value** | A | `tooltip` + describedby |

---

## 5. Checklist

- [ ] Content has `role="tooltip"` and stable id
- [ ] Trigger sets `aria-describedby` while open
- [ ] Appears on focus and hover; Escape dismisses
- [ ] No interactive content inside tooltip
- [ ] No focus trap / aria-modal
- [ ] Contrast and reduced motion OK
