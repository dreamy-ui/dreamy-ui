# Progress — Accessibility Requirements

Guide for implementing and testing accessibility of the Dreamy UI `Progress` component.

**APG note:** Use [`progressbar`](https://www.w3.org/TR/wai-aria-1.2/#progressbar) for task/loading completion — **not** the [Meter pattern](https://www.w3.org/WAI/ARIA/apg/patterns/meter/) (meters are for fixed-range measurements like battery).

---

## 1. Purpose and semantics

A progress bar communicates how far a process has advanced within a known range (or that progress is indeterminate). The widget must expose **role**, **name**, and **value** to assistive technologies.

---

## 2. Component anatomy

| Part | Element (Dreamy) | Role in a11y tree | Purpose |
| --- | --- | --- | --- |
| **Root** | `<div>` (styled track) | Presentation / container | Visual track |
| **Filled track** | `<div>` | **`role="progressbar"`** | Value + name live here |
| **Children** | optional | Text | Extra label / content |

```
div.track → div[role=progressbar][aria-valuemin|max|now][aria-label]
```

Dreamy places `role="progressbar"` and value attributes on the **filled track**; `aria-label` is required on `Progress` props.

---

## 3. Requirements

### Role and values

| Attribute | Determinate | Indeterminate |
| --- | --- | --- |
| `role` | `progressbar` | `progressbar` |
| `aria-valuemin` | min (default `0`) | min |
| `aria-valuemax` | max (default `100`) | max |
| `aria-valuenow` | current `value` | **omit** |
| `aria-valuetext` | optional human string (e.g. “3 of 4 files”) | optional status text |

- Clamp `value` within min/max for display and `aria-valuenow`.
- Indeterminate: when `value` is undefined / loading unknown — do not set `aria-valuenow`.

### Naming (required)

- Accessible name via `aria-label` (required in Dreamy props) or `aria-labelledby` pointing at visible text.
- Name describes the process (“Upload progress”), not the percentage alone.
- Visible percentage text should match or be reflected in `aria-valuenow` / `aria-valuetext` (WCAG 2.5.3 when visible).

### Visual / motion

- Progress not by color alone (fill width / determinate indicator).
- Track/fill contrast ≥ 3:1 (WCAG 1.4.11).
- Indeterminate animation: respect `prefers-reduced-motion` / Dreamy reduce-motion.

### Keyboard

- Progressbars are not typically focusable unless also a control; no required keyboard pattern.

---

## 5. Checklist

- [ ] Element exposes `role="progressbar"`
- [ ] Accessible name via `aria-label` or `aria-labelledby`
- [ ] `aria-valuemin` / `aria-valuemax` set
- [ ] Determinate: `aria-valuenow` set; indeterminate: omitted
- [ ] `aria-valuetext` when percent alone is unclear
- [ ] Visual state matches exposed value
- [ ] Reduced motion respected for indeterminate animation
