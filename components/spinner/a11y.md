# Spinner — Accessibility Requirements

Guide for implementing and testing accessibility of the Dreamy UI `Spinner` component.

**Anatomy:** root container, decorative animated circles, optional visible `label`.

---

## 1. Purpose and semantics

Spinner communicates a **loading / busy** state. It is not an interactive control.

| Mode | When | Requirement |
| --- | --- | --- |
| **Status** | Loading is meaningful and should be announced | Expose status semantics + accessible name |
| **Decorative** | Adjacent text already explains loading | Hide from AT (`aria-hidden="true"`) |

---

## 2. Requirements

### Status / busy (default when `label` is provided)

- Prefer `role="status"` on the root (or an equivalent live region) so AT can announce loading without focus moves.
- Provide an accessible name via visible `label` and/or `aria-label` / `aria-labelledby` (e.g. “Loading”).
- Animated circles are decorative — keep them out of the name (no alternate text on `<i>` parts).
- Do **not** put focus on the spinner; do **not** trap or block keyboard users while content loads (WCAG 2.1.2). Busy UI may disable specific controls, but must not freeze the whole page keyboard path.

### Decorative

- When a parent already has “Loading…” text (or `aria-busy` on a region), set `aria-hidden="true"` on Spinner so the animation is not announced twice.
- Still allow optional visible `label` for sighted users if the product wants it — but if the label is visible, prefer status mode instead of hiding the whole tree.

### Motion

- Respect `prefers-reduced-motion` / Dreamy reduced-motion settings for the spin animation (WCAG 2.3.3).

---

## 3. WCAG 2.2 (relevant)

| SC | Level | Why |
| --- | --- | --- |
| **1.1.1 Non-text Content** | A | Spinner graphic needs a text alternative when it conveys status |
| **1.4.11 Non-text Contrast** | AA | Visible spinner vs background ≥ 3:1 when meaningful |
| **2.1.2 No Keyboard Trap** | A | Loading must not trap focus |
| **2.2.2 Pause, Stop, Hide** | A | Purely decorative animation must not distract indefinitely without a way to stop/hide (prefer reduced motion + short-lived status) |
| **4.1.3 Status Messages** | AA | Loading status should be available to AT without moving focus |

---

## 4. Checklist

- [x] Meaningful spinner has a name (`label` / `aria-label`) and status semantics
- [x] Decorative spinner is `aria-hidden`
- [x] Circles are not separately announced
- [x] Focus is not forced onto the spinner; keyboard path remains usable
- [x] Reduced motion respected
