# Transitions — Accessibility Requirements

Guide for implementing and testing accessibility of Dreamy UI transition helpers (`Collapse`, `Scale`).

**Not a primary widget.** These wrap content with enter/exit motion. Accessibility belongs to the **content and controlling pattern** (disclosure, dialog, menu, etc.), not to the motion wrapper itself.

---

## 1. Purpose and semantics

| Export | Role |
| --- | --- |
| `Collapse` | Height/opacity show-hide animation |
| `Scale` | Scale/opacity show-hide animation |

- Do **not** add widget roles (`button`, `dialog`, `region`, …) to the motion root unless the consumer’s pattern requires them.
- Visibility / open state must remain programmatically determinable via the controlling pattern (`isOpen`, `aria-expanded`, `hidden`, unmount, etc.) — animation alone is not enough.
- Prefer [APG Disclosure](https://www.w3.org/WAI/ARIA/apg/patterns/disclosure/), [Dialog](https://www.w3.org/WAI/ARIA/apg/patterns/dialog-modal/), or the relevant pattern for the host control.

---

## 2. Reduced motion (required)

Dreamy already wires `useReducedMotion` + `transformReducedMotion` in both components.

| Requirement | Detail |
| --- | --- |
| Honor `prefers-reduced-motion` | Instant or near-instant enter/exit; no large scale/height motion |
| Honor Dreamy `reduceMotion` | Same as OS preference when set globally |
| Content still usable | Open/close and focus behavior must work with motion disabled |

Mapped to [WCAG 2.3.3 Animation from Interactions](https://www.w3.org/WAI/WCAG22/Understanding/animation-from-interactions.html) (AAA) and [2.2.2 Pause, Stop, Hide](https://www.w3.org/WAI/WCAG22/Understanding/pause-stop-hide.html) when motion is continuous.

---

## 3. Content and focus

- Do not trap focus inside a transitioning wrapper.
- If `unmountOnExit` removes content, ensure focus moves to a sensible place before unmount (host pattern owns this).
- Collapsed / exited content that remains in the DOM must not be reachable when closed (e.g. `inert`, `hidden`, or unmounted) — host pattern owns this.

---

## 4. WCAG 2.2 (relevant)

| SC | Level | Why |
| --- | --- | --- |
| **2.2.2 Pause, Stop, Hide** | A | Avoid endless decorative animation |
| **2.3.3 Animation from Interactions** | AAA | Respect reduced motion for enter/exit |
| **4.1.2 Name, Role, Value** | A | Open/closed state comes from the host pattern, not CSS animation |

Also keep contrast of revealed content ([1.4.3](https://www.w3.org/WAI/WCAG22/Understanding/contrast-minimum.html), [1.4.11](https://www.w3.org/WAI/WCAG22/Understanding/non-text-contrast.html)).

---

## 5. Checklist

- [ ] No widget role on the motion wrapper unless the host pattern needs it
- [ ] `prefers-reduced-motion` / Dreamy `reduceMotion` shorten or disable motion
- [ ] Open/close still works with reduced motion
- [ ] Closed content is not focusable / operable
- [ ] Focus management owned by the host pattern, not broken by exit animation
