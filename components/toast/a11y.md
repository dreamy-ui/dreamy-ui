# Toast — Accessibility Requirements

Guide for implementing and testing accessibility of the Dreamy UI `Toast` component.

**APG / ARIA:** [Alert](https://www.w3.org/WAI/ARIA/apg/patterns/alert/), [status role](https://www.w3.org/TR/wai-aria-1.2/#status), WCAG **4.1.3 Status Messages**

---

## 1. Purpose and semantics

A toast is a **transient status message**. It must be announced to assistive technologies without moving focus (unless the product intentionally focuses a critical toast).

| Severity / intent | Preferred live semantics |
| --- | --- |
| Errors / urgent | `role="alert"` (implicit `aria-live="assertive"`) |
| Info / success / loading | `role="status"` (implicit `aria-live="polite"`) |

Do **not** confuse toasts with modal `alertdialog` — toasts do not trap focus or block the page.

---

## 2. Component anatomy

| Part | Purpose in a11y |
| --- | --- |
| **Toast root** (`StyledToast` / custom `render`) | Live region host |
| **Title** (`data-part="title"`) | Primary message |
| **Description** (`data-part="description"`) | Secondary detail |
| **Icon** / Spinner | Decorative or labelled; avoid duplicate announcement |
| **CloseButton** (`data-part="close"`) | Optional dismiss; must be named |
| **rightContent** | Optional actions — keep minimal; ensure keyboard reachability |

Rendered inside `ToastProvider` / `ToastManager` portals by position.

---

## 3. Requirements

### Live region

- Each toast (or its queue container) must expose an appropriate **live region** so new toasts are announced (WCAG 4.1.3).
- Prefer polite for non-critical; assertive only when interruption is justified.
- Avoid announcing purely decorative icons (`aria-hidden` on status glyphs when the title already conveys meaning).

### Interaction

- Closable toasts: named Close control; operable by keyboard.
- Auto-dismiss: pause on hover/focus when possible so users can read (aligns with 2.2.1 timing considerations).
- Infinite duration toasts that convey critical errors should remain until dismissed.
- Do not steal focus on appear by default.

### Motion

- Respect `prefers-reduced-motion` / Dreamy reduce-motion for enter/exit animation.

## 4. WCAG (relevant)

| SC | Level | Notes |
| --- | --- | --- |
| **4.1.3 Status Messages** | AA | Announced via live region / role |
| **2.2.1 Timing Adjustable** | A | Auto-dismiss must be pauseable/extendable when feasible |
| **2.1.1 Keyboard** | A | Close / actions |
| **1.4.1 Use of Color** | A | Status not by color alone (icon + text) |
| **4.1.2 Name, Role, Value** | A | Close control named |

---

## 5. Checklist

- [x] Toast uses `role="status"` or `role="alert"` as appropriate
- [x] New toasts are announced without requiring focus move
- [x] Icons do not create redundant / noisy announcements
- [x] CloseButton has an accessible name when `isClosable`
- [x] Auto-dismiss can be paused (e.g. hover) when duration is finite
- [x] Not implemented as a focus-trapping modal dialog
