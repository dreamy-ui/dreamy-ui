# Image — Accessibility Requirements

Guide for implementing and testing accessibility of the Dreamy UI `Image` component.

**Focus:** text alternatives; decorative vs informative. Align with [WCAG 1.1.1](https://www.w3.org/WAI/WCAG22/Understanding/non-text-content.html) and HTML `alt` guidance.

---

## 1. Purpose and semantics

`Image` wraps a native `<img>` (optional zoom/blur wrappers). Accessibility is almost entirely about the **correct `alt` (or equivalent)** and not exposing decorative duplicates.

---

## 2. Component anatomy

| Part | Element (Dreamy) | Role in a11y tree | Purpose |
| --- | --- | --- | --- |
| **Image** | `<img>` | Image | Primary graphic |
| **Wrapper** | `<div>` (zoom / blur modes) | Presentation | Layout / effects only |
| **Blurred clone** | second `<img aria-hidden>` | Hidden | Visual shadow; must not be announced |

```
img[alt] 
  or  div.wrapper → img[alt] + img[aria-hidden] (blurShadow)
```

---

## 3. Requirements

### Informative vs decorative

| Use case | `alt` / exposure |
| --- | --- |
| **Informative** (content, diagram, product photo that conveys meaning) | Meaningful `alt` describing purpose/content — not “image of…” |
| **Decorative** (purely visual, adjacent text already explains) | `alt=""` so AT skips it |
| **Functional** (image is the only label of a control) | `alt` names the action/destination; or name the wrapping control |

### Naming rules

- Authors must pass an appropriate `alt` (Dreamy does not invent one).
- Empty `alt=""` is correct for decorative images; omitting `alt` is incorrect for informative images.
- Complex images (charts): short `alt` + longer description (`aria-describedby` / adjacent text / caption).
- Fallback `src` (`fallbackSrc`) must not change the meaning of `alt`.

### Effects wrappers

- Zoom wrapper: not a second image in the a11y tree.
- Blur shadow clone: always `aria-hidden="true"` (Dreamy default).
- Motion (zoom on hover): respect `prefers-reduced-motion` where animation is significant (WCAG 2.3.3).

### Keyboard

- Static images are not focusable.
- Linked/button images: focus on the control; ensure visible focus.

## 4. WCAG 2.2 (relevant)

| SC | Level | Why |
| --- | --- | --- |
| **1.1.1 Non-text Content** | A | Correct `alt` |
| **1.4.5 Images of Text** | AA | Prefer real text over text-in-image |
| **1.4.11 Non-text Contrast** | AA | When image is a UI affordance |
| **2.3.3 Animation from Interactions** | AAA | Zoom hover |
| **4.1.2 Name, Role, Value** | A | Image name when informative |

---

## 5. Checklist

- [x] Informative images have meaningful `alt`
- [x] Decorative images use `alt=""`
- [x] Blur duplicate is `aria-hidden`
- [x] No double announcement of the same graphic
- [x] Complex images have extended description when needed
- [x] Interactive image controls are keyboard operable with visible focus
