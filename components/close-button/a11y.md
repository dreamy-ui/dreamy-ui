# CloseButton — Accessibility Requirements

Guide for implementing and testing accessibility of the Dreamy UI `CloseButton` component.

**Primary APG reference:** [Button Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/button/)

CloseButton is an **IconButton** preset for dismiss/close actions. Default `aria-label="Close"`.

---

## 1. Purpose and semantics

A compact control that dismisses a surface (modal, popover, toast, sidebar, etc.).

```tsx
<CloseButton aria-label="Close dialog" onClick={onClose} />
```

- Role: button (via IconButton → Button).
- Accessible name required; default is `"Close"` — **override** when the context is ambiguous (“Close settings”, “Dismiss notification”).

---

## 2. Component anatomy

| Part | Element (Dreamy) | Role in a11y tree | Purpose |
| --- | --- | --- | --- |
| **Root** | `<button>` via IconButton | Button | Focus + activation |
| **Default SVG** | close “X” icon | May expose `role="img"` + `aria-label` | Visual affordance |

**Naming caution:** the default SVG currently sets `role="img"` and `aria-label` matching the button. Prefer a single name on the **button** and mark the SVG decorative (`aria-hidden="true"`) to avoid double announcements. Treat that as the accessibility target when fixing/implementing.

---

## 3. Requirements

### Accessible name

| Rule | Detail |
| --- | --- |
| Default | `aria-label="Close"` if unspecified |
| Contextual labels | Override whenever multiple close controls exist or the surface name matters |
| i18n | Translate the label |

### Disabled state

- Support `isDisabled` / `disabled` like Button — not operable when disabled.
- Do not disable the only way to dismiss a modal unless another accessible dismiss path exists (Escape + documented alternative).

### Keyboard

| Key | Requirement |
| --- | --- |
| **Tab** | Focus the close control (modals: usually included in the focus loop) |
| **Enter** / **Space** | Activates close |
| **Escape** | On dialogs/popovers, should also dismiss (owned by overlay pattern; CloseButton remains available) |

Visible focus required. Target size ≥ 24×24 CSS px even when visually `sm`.

### Composition with overlays

- Closing must restore focus to the control that opened the surface (dialog/popover pattern).
- CloseButton must remain reachable while the overlay is open (no inert/hidden without an alternative).

---

## 4. WCAG highlights

| SC | Why |
| --- | --- |
| **1.1.1 / 4.1.2** | Icon-only control named |
| **2.1.1 / 2.1.2 / 2.4.7** | Keyboard dismiss; no trap; focus visible |
| **2.5.8** | Minimum target size |

---

## 5. Checklist

- [x] Accessible name present (default or contextual override)
- [x] Icon does not cause duplicate AT announcements
- [x] Enter / Space activate; disabled not operable
- [x] Usable inside modal/popover focus order
- [x] Focus visible; adequate target size
