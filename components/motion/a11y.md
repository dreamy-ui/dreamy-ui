# Motion — Accessibility

`MotionBox` and `MotionFlex` are animated layout wrappers (`motion` + styled `div`). They are **not** APG interactive widgets. Accessibility focus is reduced motion, not conveying meaning by animation alone, and not breaking child semantics.

---

## Semantics

- Same as `Box` / `Flex`: generic containers unless you add a role.
- Animation props affect presentation only — they must not be the only way users perceive state or feedback.
- Prefer animating transform/opacity; avoid large flashing or seizure-risk patterns.

---

## Reduced motion

Respect user motion preferences (typically `prefers-reduced-motion: reduce`):

- Disable or shorten non-essential entrance/exit/looping animations.
- Keep essential UI available instantly without waiting on motion.
- Do not require hovering or watching an animation to understand content.

Library usage should gate decorative motion when reduced-motion is set (CSS and/or Motion’s reduced-motion options).

---

## Authoring checklist

| Do | Don't |
| --- | --- |
| Pair motion with durable UI state (text, icons, attributes) | Convey success/error **only** by a color flash or bounce |
| Offer an equivalent experience with motion off | Autoplay large repeating animations |
| Keep focus and reading order independent of animation | Move focus solely via animated position changes |
| Test with OS “reduce motion” enabled | Use continuous flashing content |

---

## WCAG notes

| Criterion | Relevance |
| --- | --- |
| **2.3.3** Animation from Interactions | Motion triggered by interaction can be disabled |
| **2.2.2** Pause, Stop, Hide | Moving content that lasts >5s needs a way to pause/stop (or don’t auto-run) |
| **2.3.1** Three Flashes | Avoid flashing above thresholds |
| **1.3.1** / **1.4.1** | Do not rely on motion or color alone for meaning |
