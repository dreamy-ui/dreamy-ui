# Editable — Accessibility Requirements

Inline edit widget: **preview** mode shows the value; **edit** mode reveals an input (and optional edit/submit/cancel buttons). Not a native single control — compose preview + textbox + buttons.

**APG:** No dedicated Editable pattern. Combine [Button](https://www.w3.org/WAI/ARIA/apg/patterns/button/) and native textbox practices from [APG Patterns](https://www.w3.org/WAI/ARIA/apg/patterns/).

---

## Dreamy form composition

When used as a form value, wrap in **`Field`** for the persistent field label/hint/error. Editable owns preview/edit chrome only — do not duplicate Field.Label into Editable.

Wire Field descriptions/invalid to the **editing input** when in edit mode (and ensure preview still has an accessible name in view mode).

```tsx
<Field>
  <Field.Label>Display name</Field.Label>
  <Editable.Root defaultValue="Ada">
    <Editable.Preview />
    <Editable.Input />
    <Editable.EditButton />
    <Editable.SubmitButton />
    <Editable.CancelButton />
  </Editable.Root>
</Field>
```

---

## Anatomy

| Part | Component | A11y role |
| --- | --- | --- |
| **Root** | `Editable.Root` | Container; may reflect editing state via `data-*` |
| **Preview** | `Editable.Preview` | Read-only presentation; focusable when `isPreviewFocusable` |
| **Input** | `Editable.Input` | Native textbox in edit mode |
| **EditButton** | `Editable.EditButton` | Button — enter edit (`aria-label` default “Edit”) |
| **SubmitButton** | `Editable.SubmitButton` | Button — commit (“Submit”) |
| **CancelButton** | `Editable.CancelButton` | Button — discard (“Cancel”) |

Only one of Preview or Input should be interactive/visible at a time (`hidden` on the inactive part).

### Connections

```
Field.Label ──► names the editable value
Preview focus / EditButton ──► enter edit mode
Input ──► value while editing; Escape cancel / Enter submit (per hook)
Submit / Cancel ──► commit or revert; return focus appropriately
```

---

## Requirements

### Modes

| Mode | Requirement |
| --- | --- |
| **View** | Preview shows current value (or placeholder); not a silent empty node |
| **Edit** | Input visible and focused; preview hidden |
| **Disabled** | `aria-disabled` / `disabled`; no edit entry |

### Naming & buttons

- Field.Label (or `aria-label` on root/preview) names the value being edited.
- Icon-only action buttons **must** keep accessible names (`aria-label` defaults: Edit / Submit / Cancel); visible text may override.
- Placeholder on empty preview/input is not a substitute for Field.Label.

### Keyboard

| Key / action | Behavior |
| --- | --- |
| Focus preview / activate Edit | Enter edit mode (unless double-click-only mode) |
| **Enter** (in input) | Submit (when enabled for single-line) |
| **Escape** | Cancel edit; restore previous value |
| **Tab** | Move among input and action buttons; no trap |

On submit/cancel, restore focus to a sensible target (preview or Edit button). Double-click-to-edit mode must still offer a keyboard path (Edit button).

### Focus & presence

- Do not leave both preview and input focusable simultaneously.
- Announce mode changes only if needed; prefer stable Field labeling over noisy live regions.
- Disabled state must block edit/submit/cancel.

## Checklist

- [ ] Field (or aria-label) names the editable value
- [ ] Preview and input are mutually exclusive in interaction
- [ ] Edit / Submit / Cancel have accessible names
- [ ] Keyboard path exists even if double-click edit is enabled
- [ ] Enter submits; Escape cancels (per configured behavior)
- [ ] Focus returns after submit/cancel
- [ ] Disabled blocks all edit actions
- [ ] Invalid/hint from Field apply to the input when editing
- [ ] Visible focus indicators on preview, input, and buttons
