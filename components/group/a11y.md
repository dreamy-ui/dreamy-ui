# Group — Accessibility

`Group` is a layout/visual wrapper for adjacent controls (shared borders, spacing, stacking). It is **not** an APG interactive widget by default. Accessibility is mostly optional grouping semantics plus not breaking children.

---

## Semantics

- Renders a `div` with flex/group styles. **No default `role`.**
- Marks children with `data-group-item` / first-last metadata for styling only — that does **not** create ARIA relationships.
- Children keep their own roles (button, input, etc.).

---

## When to use `role="group"`

Add `role="group"` (and a name) when the set of controls is a meaningful unit that should be announced together:

```tsx
<Group role="group" aria-label="Text alignment">
  <IconButton aria-label="Align left" />
  <IconButton aria-label="Align center" />
  <IconButton aria-label="Align right" />
</Group>
```

| Situation | Guidance |
| --- | --- |
| Related icon buttons / segmented actions | Prefer `role="group"` + `aria-label` or `aria-labelledby` |
| Form controls that already have a `<fieldset>` / `Fieldset` | Prefer native fieldset/legend; avoid duplicate grouping roles |
| Pure visual attachment (borders only) | No role required |
| Single child | No group role needed |

Do **not** use `role="group"` without an accessible name when the group itself is meant to be perceived.

---

## Authoring checklist

| Do | Don't |
| --- | --- |
| Name each interactive child | Rely on visual attachment alone for meaning |
| Use `role="group"` + label for related tool-like sets | Nest unlabeled groups that add noise |
| Preserve focus order of children | Remove or override child labels for styling |

---

## WCAG notes

| Criterion | Relevance |
| --- | --- |
| **1.3.1** Info and Relationships | Explicit group naming when relationship matters |
| **2.4.3** Focus Order | Children remain separately focusable in DOM order |
| **4.1.2** Name, Role, Value | Named `role="group"` when you expose it |
