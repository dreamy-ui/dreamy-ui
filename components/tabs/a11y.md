# Tabs — Accessibility Requirements

Guide for implementing and testing accessibility of the Dreamy UI `Tabs` component.

**APG pattern:** [Tabs](https://www.w3.org/WAI/ARIA/apg/patterns/tabs/)

---

## 1. Purpose and semantics

Tabs show one **tab panel** at a time. A `tablist` contains `tab` controls; each selected tab is associated with a `tabpanel`.

---

## 2. Component anatomy

| Part | Purpose in a11y |
| --- | --- |
| **`Tabs.Root`** | Selection state, id prefix, orientation |
| **`Tabs.List`** | `role="tablist"` (+ `aria-orientation`) |
| **`Tabs.Tab`** | `role="tab"`; `aria-selected`, `aria-controls` |
| **`Tabs.Panels`** | Panels wrapper |
| **`Tabs.Panel`** | `role="tabpanel"`; `aria-labelledby` → tab |
| **Indicator** (internal) | Visual only; decorative |

---

## 3. Requirements

### Roles and state

- List: `role="tablist"`; set `aria-orientation` to `horizontal` or `vertical` to match layout.
- Tab: `role="tab"`, `aria-selected`, `aria-controls` → panel id; disabled tabs not selectable.
- Panel: `role="tabpanel"`, `aria-labelledby` → tab id; only the active panel visible (or others `hidden`).
- **Roving tabindex**: one tab in the tab stop sequence (`tabIndex={0}`); others `-1` until focused via arrows.

### Keyboard (APG)

| Key | Requirement |
| --- | --- |
| **Tab** | Into tablist (selected/focused tab), then into the active panel’s focusables |
| **ArrowLeft** / **ArrowRight** | Horizontal: previous / next tab (RTL-aware) |
| **ArrowUp** / **ArrowDown** | Vertical orientation |
| **Home** / **End** | Optional: first / last tab |
| **Enter** / **Space** | Activate focused tab when using manual activation |

Automatic activation (focus moves selection) vs manual (`isManual`) must stay consistent and predictable (3.2.1 / 3.2.5 considerations).

### Indicator

- Visual selection affordance only; must not replace `aria-selected`.
- Respect reduced motion for layout animation.

---

## 5. Checklist

- [ ] `tablist` / `tab` / `tabpanel` roles present
- [ ] `aria-selected` and `aria-controls` / `aria-labelledby` linked
- [ ] Roving tabindex among tabs
- [ ] Arrow keys move between tabs (orientation-aware)
- [ ] Inactive panels hidden from interaction
- [ ] Focus visible; indicator animation respects reduced motion
