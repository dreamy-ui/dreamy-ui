# Text — Accessibility Requirements

Guide for implementing and testing accessibility of the Dreamy UI `Text` component.

**Anatomy:** styled paragraph by default (`dreamy.p`). Polymorphic `as` may render other text-level elements.

---

## 1. Purpose and semantics

`Text` is for **body / supporting copy**. It is not a heading control and must not replace the document outline.

---

## 2. Requirements

### Semantics

- Default `<p>` is appropriate for paragraphs.
- Use `as` for inline/other text roles (`span`, `label`, etc.) when markup requires it.
- Do **not** use large/bold `Text` as a fake heading — use `Heading` (or `h1`–`h6`) instead (WCAG 1.3.1 / 2.4.6).

### Readability

- Meet text contrast: **4.5:1** normal, **3:1** large (WCAG 1.4.3).
- Do not convey meaning by color alone (WCAG 1.4.1); pair with text/icons.
- Ensure content reflows and remains readable at zoom / text resize (WCAG 1.4.4 / 1.4.10).

### Interactive text

- Clickable text that navigates must be a `Link` (`<a href>`), not a `Text` + `onClick`.
- Clickable text that performs an action must be a `button` (or similar), with keyboard support and focus styles.

## 3. WCAG 2.2 (relevant)

| SC | Level | Why |
| --- | --- | --- |
| **1.3.1 Info and Relationships** | A | Paragraph vs heading semantics |
| **1.4.1 Use of Color** | A | Meaning not color-only |
| **1.4.3 Contrast (Minimum)** | AA | Text contrast |
| **1.4.4 Resize Text** | AA | Remains readable when resized |
| **1.4.10 Reflow** | AA | No loss of content at narrow widths |
| **2.4.6 Headings and Labels** | AA | Don’t misuse Text as headings |

---

## 4. Checklist

- [x] Body copy uses `Text`; titles use `Heading`
- [x] No fake headings via bold/size alone
- [x] Contrast meets AA
- [x] Interactive behaviors use link/button components, not clickable `Text`
