# IconButton — Accessibility Requirements

Guide for implementing and testing accessibility of the Dreamy UI `IconButton` component.

**Primary APG reference:** [Button Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/button/)

IconButton is a **Button** specialized for icon-only actions. The TypeScript API **requires** `aria-label`.

---

## 1. Purpose and semantics

An icon-only control that performs an action. Same role and keyboard model as Button; the accessible name cannot come from visible text children alone in typical usage.

```tsx
<IconButton aria-label="Edit profile" icon={<EditIcon />} />
```

---

## 2. Component anatomy

| Part | Element (Dreamy) | Role in a11y tree | Purpose |
| --- | --- | --- | --- |
| **Root** | `<button>` via `Button` | Button | Focus + activation |
| **Icon** / `children` | graphic | Decorative content | Visual affordance; name from `aria-label` |

---

## 3. Requirements

### Accessible name (required)

| Rule | Detail |
| --- | --- |
| **`aria-label` required** | Prop is mandatory in `IconButtonProps` |
| Name quality | Describe the **action** (“Delete message”), not the glyph (“Trash icon”) |
| Locale | Translate `aria-label` with the UI language |
| `aria-labelledby` | Allowed via HTML props if pointing at visible text elsewhere; still ensure a name exists |

Do not ship icon-only buttons with empty or generic labels (“button”, “icon”).

### Icon content

- Prefer decorative treatment so the name is not duplicated awkwardly (icon `aria-hidden` when the button already has `aria-label`).
- If an SVG has its own `aria-label` / `role="img"`, avoid **double naming** (button + image both announcing).

### Disabled and loading

- Same as Button: `isDisabled` / `disabled` / `isLoading` → not operable; native `disabled` when inactive.
- Loading: keep a stable, meaningful `aria-label` (or provide `loadingText` patterns via Button).

### Keyboard

| Key | Requirement |
| --- | --- |
| **Tab** | Focus the control |
| **Enter** / **Space** | Activate |
| Disabled / loading | No activation |

Visible focus required. Hit target ≥ 24×24 CSS px (2.5.8) — icon buttons are often small; pad the control, not only the SVG.

---

## 4. WCAG highlights

| SC | Why |
| --- | --- |
| **1.1.1** | Icon meaning conveyed via accessible name |
| **2.1.1 / 2.4.7** | Keyboard + focus visible |
| **2.5.8** | Target size |
| **4.1.2** | Name, button role, disabled state |

---

## 5. Checklist

- [ ] `aria-label` present and action-oriented
- [ ] No dual conflicting names (button + labeled SVG)
- [ ] Disabled / loading not activatable
- [ ] Enter / Space work; focus visible
- [ ] Target size ≥ 24×24 CSS px
