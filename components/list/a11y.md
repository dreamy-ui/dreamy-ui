# List — Accessibility Requirements

Guide for implementing and testing accessibility of the Dreamy UI `List` component.

**Related:** native HTML list semantics; [APG patterns index](https://www.w3.org/WAI/ARIA/apg/patterns/) (no dedicated list widget pattern).

---

## 1. Purpose and semantics

Lists group related items. Choose **semantic list** (`ul`/`ol` + `li`) when order or grouping matters to meaning; use **presentational** markup when items are only visual layout and must not be announced as a list.

---

## 2. Component anatomy

| Part | Element (Dreamy) | Role in a11y tree | Purpose |
| --- | --- | --- | --- |
| **Root** | `<ul>` or `<ol>` (`ordered` / default unordered) | List | Container |
| **Item** | `<li>` | List item | One entry |

```
ul|ol → li → …
```

Dreamy: `ordered` → `<ol>`; otherwise `<ul>`. Do not set both `ordered` and `unordered`.

---

## 3. Requirements

### Semantic list (default)

- Use `Root` + `Item` so AT announces list and item count.
- Ordered sequences (rankings, steps as content, numbered instructions) → `ordered`.
- Unordered collections → default / `unordered`.
- Nest lists correctly with nested `ul`/`ol` inside `li`.

### Presentational (when not a “list” of meaning)

When a visual stack is **not** a list of related items (e.g. purely decorative grouping):

- Prefer non-list layout primitives (`Stack`, `Flex`), **or**
- Keep structure but neutralize with care: `role="presentation"` / `role="none"` on list and items only when removing list semantics is intentional and children remain exposed correctly.

Do **not** strip list semantics from real content lists just for styling.

### Content

- List items may contain text, links, or controls; each interactive child needs its own name and keyboard support.
- Icons before items: decorative → `aria-hidden`; informative → text alternative.

### Keyboard

- Lists themselves are not widgets; Tab moves through focusable descendants in DOM order.

---

## 5. Checklist

- [x] Related items use `ul`/`ol` + `li` (Dreamy Root/Item)
- [x] Ordered content uses `ordered` → `<ol>`
- [x] Layout-only stacks do not falsely announce as lists
- [x] Decorative icons hidden; informative icons named
- [x] Nested lists nested inside `li`
- [x] Focusable children work with keyboard
