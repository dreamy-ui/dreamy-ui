# Fieldset — Accessibility Requirements

Guide for implementing and testing accessibility of the Dreamy UI `Fieldset` component.

**Fieldset owns group chrome** for related controls. Per-control label / hint / error remains **`Field`’s job**. Fieldset provides the **group name** (legend) and optional group-level help/error.

**Primary references:** native [`<fieldset>` / `<legend>`](https://html.spec.whatwg.org/multipage/form-elements.html#the-fieldset-element); [WCAG 1.3.1 Info and Relationships](https://www.w3.org/WAI/WCAG22/Understanding/info-and-relationships.html); [WCAG 3.3.2 Labels or Instructions](https://www.w3.org/WAI/WCAG22/Understanding/labels-or-instructions.html).

---

## 1. Purpose and semantics

Fieldset groups related form controls under one caption so assistive technologies announce the **group context** together with each control’s own name.

```tsx
<Fieldset.Root>
  <Fieldset.Header>
    <Fieldset.Legend>Shipping address</Fieldset.Legend>
    <Fieldset.HelperText>Where should we deliver?</Fieldset.HelperText>
  </Fieldset.Header>
  <Fieldset.Content>
    <Field label="Street"><Input /></Field>
    <Field label="City"><Input /></Field>
  </Fieldset.Content>
  <Fieldset.Footer>{/* actions */}</Fieldset.Footer>
</Fieldset.Root>
```

---

## 2. Component anatomy (multipart)

| Part | Element (Dreamy) | Role in a11y tree | Purpose |
| --- | --- | --- | --- |
| **Root** | `<fieldset>` | Group | Native grouping; `disabled` disables descendants |
| **Legend** | `<legend>` | Group name | **Required** accessible name for the fieldset |
| **HelperText** | `<div>` | Description | Group-level instructions |
| **ErrorText** | `<div>` | Error description | Group-level error (still prefer text, not color alone) |
| **Header** | `<div>` | Presentational layout | Wraps legend + helper |
| **Content** | `<div>` | Presentational layout | Wraps Fields / controls |
| **Footer** | `<div>` | Presentational layout | Actions (submit, etc.) |

---

## 3. Division of responsibility

| Concern | Fieldset | Field | Control (Input, Switch, …) |
| --- | --- | --- | --- |
| Group name | **Legend** | — | — |
| Group hint / error | HelperText / ErrorText | — | — |
| Per-control label | — | **Label** | Consumes `id` / labelledby |
| Per-control hint / error | — | **Hint** / **Error** | `aria-describedby` / `aria-invalid` |
| Widget role/value | — | — | Control owns semantics |

**Do not** replace `Field.Label` with only a Fieldset legend when each control needs its own name. Legend names the **set**; Field labels each **member**.

---

## 4. Per-part requirements

### Root (`<fieldset>`)

- Use real `<fieldset>` (Dreamy does) — prefer native over `role="group"` unless there is a strong reason.
- `disabled`: native attribute disables included controls; expose visual `data-disabled`.
- `invalid` / `aria-invalid`: group-level invalid cue; still provide **ErrorText** (and/or Field errors). Do not rely on `aria-invalid` alone without message text (3.3.1).

### Legend

- First meaningful caption for the group; visible text preferred.
- Keep legend concise; put long instructions in HelperText.
- Avoid empty or decorative-only legends.

### HelperText / ErrorText

- Visible text for instructions / errors.
- When linking to the group for AT, use `aria-describedby` on the fieldset pointing at helper/error ids (if not already implied by structure).
- ErrorText: not color-only; hide or replace helper when showing a group error if that matches product UX.

### Header / Content / Footer

- Layout only; no extra widget roles.
- Footer buttons need their own accessible names (Button rules).

---

## 5. Keyboard and focus

- Fieldset itself is not a tab stop.
- Tab order follows included Fields/controls/footer actions logically (2.4.3).
- Disabling the fieldset must make included controls non-operable.

---

## 6. WCAG highlights

| SC | Why |
| --- | --- |
| **1.3.1** | Group relationship via fieldset/legend |
| **3.3.1 / 3.3.2** | Group and per-field labels/errors |
| **4.1.2** | Group name determinable; controls keep their own name/role/value |

---

## 7. Checklist

- [x] Root is `<fieldset>` with a real **`<legend>`**
- [x] Each control still wrapped in **Field** (or otherwise individually labeled)
- [x] Group helper/error are text, not color-only
- [x] `disabled` on fieldset disables controls
- [x] Group `aria-invalid` paired with ErrorText when used
- [x] No duplicate/conflicting group roles on layout slots
