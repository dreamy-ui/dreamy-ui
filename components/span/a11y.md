# Span — Accessibility

`Span` is an inline text/layout primitive (`span`). It is **not** an APG interactive widget. Accessibility is about keeping inline semantics correct and not turning spans into fake controls.

---

## Semantics

- Renders a generic inline element. No default ARIA role.
- Suitable for styling fragments of text; not a substitute for headings, links, or buttons.
- Children/text remain part of the surrounding paragraph or parent’s accessible name when composed correctly.

---

## Authoring checklist

| Do | Don't |
| --- | --- |
| Style inline text, icons wrappers (with care), or fragments | Use `Span` with `onClick` as a button without role + keyboard + name |
| Prefer `<a>` / `<button>` for navigation and actions | Split sentences across spans in ways that break screen-reader phrasing unnecessarily |
| Mark decorative icons `aria-hidden="true"` | Put block-level interactive widgets inside a span when a `div`/semantic element is clearer |

---

## WCAG notes

| Criterion | Relevance |
| --- | --- |
| **1.3.1** Info and Relationships | Inline styling must not replace semantic elements |
| **1.3.2** Meaningful Sequence | Keep text order natural in the DOM |
| **4.1.2** Name, Role, Value | Required only if you add an interactive role |
