# Checkbox â€” Accessibility Requirements

Binary or tri-state form control for selecting options. Prefer the native checkbox architecture used by Dreamy (`input[type="checkbox"]` + visual control).

**APG:** [Checkbox Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/checkbox/)

---

## Dreamy form composition

Dreamy checkboxes are **standalone**. Compose them **inside `Field`**.

| Concern | Owned by | Not owned by Checkbox |
| --- | --- | --- |
| Visible label | `Field.Label` | Do not add Field label chrome into Checkbox |
| Hint / error | `Field.Hint` / `Field.Error` | Do not bake hint/error UI into Checkbox |
| `id`, `aria-describedby`, invalid/required | Field â†’ Checkbox via `useField` | Checkbox must consume Field, not reimplement it |

Optional `children` are an **inline adjacent name**, separate from `Field.Label`. Preferred:

```tsx
<Field>
  <Field.Label>Accept terms</Field.Label>
  <Checkbox />
  <Field.Error>Required</Field.Error>
</Field>
```

The control must still expose name/role/value and wire Field attributes (`id`, `aria-describedby`, `aria-invalid`, etc.).

## Anatomy

| Part | Element | A11y role |
| --- | --- | --- |
| **Root** | styled `div` | Click target / visual container (not the widget) |
| **Input** | visually hidden `input[type="checkbox"]` | **The checkbox** â€” focus, keyboard, form value |
| **Control** | wrapper `span` + icon | Decorative (`aria-hidden="true"`) |
| **Inline label** | optional `span` (`children`) | May contribute to accessible name |

Icons (`CheckboxIcon`, indeterminate) must be `aria-hidden="true"`.

## Requirements

### Role & state

- Prefer native `input[type="checkbox"]` (no custom `role="checkbox"` needed).
- Expose checked via native `checked`.
- Support **indeterminate** via the DOM `indeterminate` property; AT may announce mixed / partially checked.
- Do not put conflicting `aria-checked` on the root if the input owns state.

### Naming

| Method | When |
| --- | --- |
| `Field.Label` â†” input `id` | Preferred |
| `aria-label` / `aria-labelledby` | When no visible Field label |
| Optional `children` | Adjacent text; not a substitute for Field |

WCAG 2.5.3: accessible name should include visible label text.

### Other input attributes

| Attribute | Source |
| --- | --- |
| `id` | Field / props |
| `aria-describedby` | Field hint + error |
| `aria-invalid` | Field `isInvalid` |
| `disabled` / `readOnly` / `required` | Field / props / group |
| `name` / `value` | Form submission |

### Keyboard

| Key | Behavior |
| --- | --- |
| **Tab** | Focus the input |
| **Space** | Toggle checked |
| **Enter** | Optional toggle if implemented consistently |

Disabled / read-only must not toggle. No keyboard trap.

### CheckboxGroup

`CheckboxGroup` shares value/disabled/invalid/required context among children.

- Expose a **group** name: `role="group"` + `aria-labelledby`, or `<fieldset>` + `<legend>` (often via Field / Fieldset).
- Each checkbox keeps its own accessible name.
- Group does not replace per-control Field wiring for descriptions/errors when each option needs them.

## Checklist

- [ ] Native checkbox input is the sole focusable widget
- [ ] Visual control/icons are `aria-hidden`
- [ ] Accessible name present (Field.Label, aria-label, or children)
- [ ] Checked / indeterminate state exposed correctly
- [ ] Field wires `id`, `aria-describedby`, `aria-invalid`
- [ ] Space toggles; disabled/read-only do not
- [ ] Visible focus indicator on the control
- [ ] CheckboxGroup has an accessible group name when used
- [ ] Target size meets 2.5.8
- [ ] Color is not the only checked/invalid cue
