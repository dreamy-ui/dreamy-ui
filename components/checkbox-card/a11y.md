# Checkbox Card — Accessibility Requirements

Card-styled checkbox: large clickable surface with title/description and a visual checkbox indicator. Same semantics as Checkbox; presentation differs.

**APG:** [Checkbox Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/checkbox/)

---

## Dreamy form composition

Standalone control; compose **inside `Field`** (or a labeled group). Field owns label/hint/error chrome. `CheckboxCard.Title` / `Description` are **in-card content**, not Field chrome — do not duplicate Field.Label into the card API.

Wire Field → hidden input: `id`, `aria-describedby`, `aria-invalid`, disabled/required.

---

## Anatomy

| Part | Component | A11y role |
| --- | --- | --- |
| **Root** | `CheckboxCard.Root` | Clickable container; activates input |
| **Input** | visually hidden `input[type="checkbox"]` | **The checkbox** — focus + state |
| **Header / Title / Description** | `Header`, `Title`, `Description` | Visible name / description content |
| **Label** | `CheckboxCard.Label` | Optional in-card label association |
| **Checkbox** | `CheckboxCard.Checkbox` | Visual indicator only (`aria-hidden` control) |

`RootProvider` + external `useCheckbox` is the same a11y contract when controlled outside the Root.

### Connections

```
Field.Label / Title ──► input (id / aria-labelledby)
Field.Hint + Error ──► input (aria-describedby)
Root click / Space ──► input.toggle
Checkbox indicator ──► decorative (aria-hidden)
```

---

## Requirements

### Role & state

- Native `input[type="checkbox"]` is the widget (same as Checkbox).
- Expose `checked` / indeterminate correctly.
- Root may use `data-checked`, `data-invalid`, `data-focus-visible` for styling — not as a second AT widget.

### Naming

- Prefer Field label **or** associate Title/Label with the input (`aria-labelledby` / `htmlFor`).
- Description should be linked via `aria-describedby` when it conveys required meaning (or rely on Field.Hint).
- Icon must not create a competing accessible name.

### Keyboard & focus

| Key | Behavior |
| --- | --- |
| **Tab** | Focus the hidden input (focus ring may paint on Root) |
| **Space** | Toggle |
| **Enter** | Optional consistent toggle |

Pointer on Root must activate the same input. Disabled/read-only must not toggle.

### Groups

Usable inside `CheckboxGroup` like Checkbox. Group needs an accessible group name (`role="group"` / fieldset). Each card still needs its own name.

## Checklist

- [x] Hidden checkbox input is the only checkbox in the a11y tree
- [x] Visual checkbox slot is not a second focusable control
- [x] Title/Label (or Field) provides accessible name
- [x] Description/hint/error linked via `aria-describedby` when needed
- [x] Checked / indeterminate / invalid states exposed on input
- [x] Space toggles; focus visible on card
- [x] Grouped cards share group labeling when in CheckboxGroup
- [x] Color alone does not convey checked/invalid
