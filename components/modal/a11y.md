# Modal — Accessibility Requirements

Guide for implementing and testing accessibility of the Dreamy UI `Modal` component.

**APG pattern:** [Dialog (Modal)](https://www.w3.org/WAI/ARIA/apg/patterns/dialog-modal/)

---

## 1. Purpose and semantics

A modal dialog overlays the page and **requires** the user to interact with it (or dismiss it) before returning to the underlying content. Background content is inert while open.

Use `role="dialog"` + `aria-modal="true"`. Prefer `role="alertdialog"` only for critical confirmations that interrupt workflow (APG Alert Dialog) — that is a separate pattern from Dreamy `Alert`.

---

## 2. Component anatomy

| Part | Purpose in a11y |
| --- | --- |
| **`Modal.Root`** | Open state, portal, focus/scroll options |
| **`Modal.Overlay`** | Visual backdrop; not a focus target |
| **`Modal.Content`** | Dialog container (`role="dialog"`, `aria-modal="true"`) |
| **`Modal.Header`** | Accessible name source (`aria-labelledby`) |
| **`Modal.Body`** | Optional description (`aria-describedby`) |
| **`Modal.Footer`** | Actions (OK / Cancel, etc.) |
| **`Modal.CloseButton`** | Explicit dismiss control |

---

## 3. Requirements

### Roles and naming

- `Modal.Content`: `role="dialog"`, `aria-modal="true"`, `tabIndex={-1}`.
- Accessible **name** required: prefer `aria-labelledby` → `Modal.Header` id; else `aria-label` on Content.
- Optional `aria-describedby` → `Modal.Body` when a short primary message should be announced on open.
- Close control must have an accessible name (e.g. “Close”).

### Focus and modality

- **Focus trap** while open (default `trapFocus`); Tab / Shift+Tab cycle only inside the dialog.
- **Initial focus** on open (`autoFocus` / `initialFocusRef`) — first meaningful control, not necessarily the close button.
- **Return focus** to the trigger on close (`returnFocusOnClose` / `finalFocusRef`).
- Make background inert (`aria-hidden` / inert polyfill) so AT and keyboard users cannot reach page content behind the modal.
- **Escape** closes the dialog (unless intentionally disabled and replaced with an equivalent dismiss path).
- No keyboard trap after close (WCAG 2.1.2).

### Keyboard

| Key | Requirement |
| --- | --- |
| **Tab** / **Shift+Tab** | Cycle focus within the dialog |
| **Escape** | Dismiss (APG) |

---

## 5. Checklist

- [ ] Content has `role="dialog"` and `aria-modal="true"`
- [ ] Dialog has an accessible name (Header / `aria-label`)
- [ ] Focus moves into dialog on open; trapped while open
- [ ] Escape and CloseButton dismiss; focus returns to trigger
- [ ] Background is inert / not reachable
- [ ] Close control is named
- [ ] Focus visible; reduced motion respected for enter/exit
