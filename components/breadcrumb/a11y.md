# Breadcrumb — Accessibility Requirements

Guide for implementing and testing accessibility of the Dreamy UI `Breadcrumb` component.

**APG:** [Breadcrumb Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/breadcrumb/)

---

## 1. Purpose and semantics

A breadcrumb trail shows the user’s place in a hierarchy and links to ancestor pages. It is a **navigation landmark**, not a generic list widget.

---

## 2. Component anatomy

| Part | Element (Dreamy) | Role in a11y tree | Purpose |
| --- | --- | --- | --- |
| **Root** | `<nav>` | Navigation landmark | Contains the trail |
| **List** | `<ol>` | List | Ordered hierarchy |
| **Item** | `<li>` | List item | One crumb |
| **Link** | `<a>` | Link | Ancestor page |
| **CurrentLink** | `<span>` | Text (`aria-current="page"`) | Current page (not a link by default) |
| **Separator** | `<li>` | Presentational (`role="presentation"`, `aria-hidden`) | Visual divider only |
| **Ellipsis** | `<li>` | Presentational (`aria-hidden`) | Collapsed middle crumbs |

```
nav[aria-label] → ol → li > a | li > span[aria-current=page] | li[aria-hidden] (separator/ellipsis)
```

---

## 3. Requirements

### Landmark and name

- Root is `<nav>` (or `role="navigation"`).
- Provide an accessible name: `aria-label="Breadcrumb"` (or `aria-labelledby`). Avoid unlabeled navigation landmarks when multiple exist on the page.

### Structure

- Use an ordered list (`<ol>` / `List`) so order is programmatic (WCAG 1.3.1).
- Separators and ellipsis must not be announced as navigation targets (`role="presentation"` + `aria-hidden`).

### Current page

- Current page uses `aria-current="page"` (Dreamy `CurrentLink` sets this by default).
- Prefer a non-link element for the current page; if it is a link, still set `aria-current="page"`.

### Links

- Each ancestor crumb is a real `<a>` with a meaningful `href` and accessible name (visible text).
- Do not rely on separator glyphs alone to convey hierarchy.

### Keyboard

- APG: no special pattern beyond normal Tab to links.
- Focus order follows trail order; visible focus on links (WCAG 2.4.7).

---

## 5. Checklist

- [x] Root is a named navigation landmark (`aria-label` / `aria-labelledby`)
- [x] Trail is an `<ol>` of `<li>` items
- [x] Current page has `aria-current="page"`
- [x] Separators / ellipsis are hidden from AT
- [x] Ancestor crumbs are focusable links with clear names
- [x] Focus visible on links
