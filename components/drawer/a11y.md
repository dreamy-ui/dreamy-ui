# Drawer — Accessibility Requirements

Guide for implementing and testing accessibility of the Dreamy UI `Drawer` component.

**APG pattern:** [Dialog (Modal)](https://www.w3.org/WAI/ARIA/apg/patterns/dialog-modal/)

---

## 1. Purpose and semantics

A drawer is a modal slide-over panel that overlays the page and **requires** the user to interact with it (or dismiss it) before returning to the underlying content. Background content is inert while open.

Use `role="dialog"` + `aria-modal="true"`.

---

## 2. Component anatomy

| Part | Purpose in a11y |
| --- | --- |
| **`Drawer.Root`** | Open state, portal, placement, focus/scroll options |
| **`Drawer.Overlay`** | Visual backdrop; not a focus target |
| **`Drawer.Content`** | Dialog container (`role="dialog"`, `aria-modal="true"`) |
| **`Drawer.Header`** | Accessible name source (`aria-labelledby`) |
| **`Drawer.Body`** | Optional description (`aria-describedby`) |
| **`Drawer.Footer`** | Actions (OK / Cancel, etc.) |
| **`Drawer.CloseButton`** | Explicit dismiss control |

---

## 3. Requirements

### Roles and naming

- `Drawer.Content`: `role="dialog"`, `aria-modal="true"`, `tabIndex={-1}`.
- Accessible **name** required: prefer `aria-labelledby` → `Drawer.Header` id; else `aria-label` on Content.
- Optional `aria-describedby` → `Drawer.Body` when a short primary message should be announced on open.
- Close control must have an accessible name (e.g. “Close”).

### Focus and modality

- **Focus trap** while open (default `trapFocus`); Tab / Shift+Tab cycle only inside the drawer.
- **Initial focus** on open (`autoFocus` / `initialFocusRef`) — first meaningful control, not necessarily the close button.
- **Return focus** to the trigger on close (`returnFocusOnClose` / `finalFocusRef`).
- Make background inert (`aria-hidden` / inert polyfill) so AT and keyboard users cannot reach page content behind the drawer.
- **Escape** closes the drawer (unless intentionally disabled and replaced with an equivalent dismiss path).
- No keyboard trap after close (WCAG 2.1.2).

### Keyboard

| Key | Requirement |
| --- | --- |
| **Tab** / **Shift+Tab** | Cycle focus within the drawer |
| **Escape** | Dismiss (APG) |

---

## 5. Checklist

- [x] Content has `role="dialog"` and `aria-modal="true"`
- [x] Dialog has an accessible name (Header / `aria-label`)
- [x] Focus moves into drawer on open; trapped while open
- [x] Escape and CloseButton dismiss; focus returns to trigger
- [x] Background is inert / not reachable
- [x] Close control is named
- [x] Focus visible; reduced motion respected for enter/exit
