# Link — Accessibility Requirements

Guide for implementing and testing accessibility of the Dreamy UI `Link` component.

**Anatomy:** styled `<a>` (via `dreamy.a`). `isExternal` sets `target="_blank"`.

---

## 1. Purpose and semantics

A link **navigates** to a URL (or in-page target). It must remain distinct from a **button**, which performs an action on the current page.

| Use | Element / role |
| --- | --- |
| Navigate to a resource / route / hash | `<a href="…">` (`Link`) |
| Submit, open dialog, toggle UI | `<button>` (not `Link`) |

Do not use `Link` without `href` as a fake button. Do not use `role="button"` on a real link that navigates.

---

## 2. Requirements

### Accessible name (required)

- Name comes from child text, nested image `alt`, or `aria-label` / `aria-labelledby`.
- Icon-only links **must** have an accessible name.
- Visible link text should be included in the name (WCAG 2.5.3). Prefer descriptive text (“View invoice”) over “Click here”.

### `href`

- Always provide a valid `href` for navigational links.
- Empty or missing `href` breaks expected link semantics and keyboard behavior.

### External links (`isExternal`)

- `target="_blank"` should be paired with `rel="noopener noreferrer"` (security + tab-nabbing mitigation).
- Announce new-window behavior when it matters (visible text or accessible name), per WCAG G201 guidance.

### Keyboard / focus

| Key | Behavior |
| --- | --- |
| **Tab** | Focuses the link |
| **Enter** | Activates navigation |

- Visible focus indicator required (WCAG 2.4.7).
- Distinguish links from surrounding text (underline / contrast — not color alone, WCAG 1.4.1).

## 3. WCAG 2.2 (relevant)

| SC | Level | Why |
| --- | --- | --- |
| **1.4.1 Use of Color** | A | Link not identified by color alone |
| **2.1.1 Keyboard** | A | Enter activates |
| **2.4.4 Link Purpose (In Context)** | A | Purpose clear from text/context |
| **2.4.7 Focus Visible** | AA | Focus indicator |
| **2.5.3 Label in Name** | A | Name matches visible text |
| **2.5.8 Target Size (Minimum)** | AA | ≥ 24×24 CSS px (or spacing exception) |
| **4.1.2 Name, Role, Value** | A | Link name and role |

---

## 4. Checklist

- [ ] Renders as `<a>` with a real `href`
- [ ] Has an accessible name (text or `aria-label`)
- [ ] Not used as a button substitute
- [ ] External links: `rel` + new-window cue when needed
- [ ] Focus visible; not color-only affordance
