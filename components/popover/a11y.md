# Popover — Accessibility Requirements

Guide for implementing and testing accessibility of the Dreamy UI `Popover` component.

**APG patterns:** [Dialog (Modal)](https://www.w3.org/WAI/ARIA/apg/patterns/dialog-modal/) (non-modal / disclosure-style dialogs), [Disclosure](https://www.w3.org/WAI/ARIA/apg/patterns/disclosure/)

---

## 1. Purpose and semantics

A popover is a **non-modal** floating panel anchored to a trigger. Users can typically still interact with the rest of the page. It is **not** a modal dialog: do **not** set `aria-modal="true"` or trap focus across the whole page unless the product intentionally makes it modal.

Default click-triggered content uses `role="dialog"`. Hover-triggered content may use `role="tooltip"` — only when content is purely informational and **not** interactive (see Tooltip / Hover Card).

---

## 2. Component anatomy

| Part | Purpose in a11y |
| --- | --- |
| **`Popover.Root`** | Open state, positioning, portal options |
| **`Popover.Trigger`** | Interactive opener (`aria-expanded`, `aria-controls`, `aria-haspopup`) |
| **`Popover.Anchor`** | Positioning reference only (optional) |
| **`Popover.Content`** | Floating panel (`role="dialog"` when click-triggered) |
| **`Popover.Header`** | Accessible name (`aria-labelledby`) |
| **`Popover.Body`** | Optional description (`aria-describedby`) |
| **`Popover.Footer`** | Actions |
| **`Popover.CloseButton`** | Explicit dismiss |
| **`Popover.Arrow`** | Decorative |

---

## 3. Requirements

### Trigger

- Must be a focusable control (`button`, link, etc.).
- `aria-expanded` reflects open state; `aria-controls` points to Content id.
- `aria-haspopup="dialog"` (or appropriate token) when Content is a dialog.

### Content

- Click trigger: `role="dialog"`, named via Header / `aria-label`.
- **No** `aria-modal` for standard Popover.
- **Escape** closes (`closeOnEsc`).
- Optional close on blur / outside click — must not strand keyboard users.
- When focus moves into Content, restore focus to Trigger on close (`returnFocusOnClose`).
- Arrow is decorative (`aria-hidden` if it would add noise).

### Keyboard

| Key | Requirement |
| --- | --- |
| **Enter** / **Space** | Open/toggle from trigger (native button behavior) |
| **Escape** | Close; return focus to trigger |
| **Tab** | Move through focusable elements in Content when focused inside |

Do not create a full-page focus trap unless intentionally modal.

## 4. WCAG (relevant)

| SC | Level | Notes |
| --- | --- | --- |
| **1.4.13 Content on Hover or Focus** | AA | If shown on hover/focus: dismissible, hoverable, persistent |
| **2.1.1 Keyboard** | A | Open, navigate, dismiss by keyboard |
| **2.1.2 No Keyboard Trap** | A | Escape / Tab out when non-modal |
| **2.4.7 Focus Visible** | AA | Trigger and inner controls |
| **4.1.2 Name, Role, Value** | A | Expanded state, dialog name |

---

## 5. Checklist

- [ ] Trigger exposes `aria-expanded` / `aria-controls`
- [ ] Content has correct role and accessible name
- [ ] Escape closes; focus returns to trigger
- [ ] Not marked `aria-modal` for standard use
- [ ] CloseButton named if present
- [ ] Hover/focus disclosure meets 1.4.13 when applicable
