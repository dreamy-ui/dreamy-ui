# Button — Accessibility Requirements

Guide for implementing and testing accessibility of the Dreamy UI `Button` component.

**Primary APG reference:** [Button Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/button/)

---

## 1. Purpose and semantics

A button triggers an action or submit. Dreamy `Button` renders a native `<button>`.

- Prefer native `<button>` (already used) over `div` + `role="button"`.
- Default `type="button"` unless explicitly `submit` / `reset`.
- Accessible name is **required** (visible children text, or `aria-label` / `aria-labelledby` when needed).

---

## 2. Component anatomy

| Part | Element (Dreamy) | Role in a11y tree | Purpose |
| --- | --- | --- | --- |
| **Root** | `<button>` | Button | Focus + activation |
| **Left / right icons** | icons in spans | Decorative | `aria-hidden` / `focusable={false}` on icon elements |
| **Spinner** | loading indicator | Decorative / status | Present during `isLoading`; name should remain clear |
| **Ripple** | visual effect | Ignored | Not announced |

Single-part control — no Field chrome. Buttons in forms are actions, not labeled Field widgets (unless used as a weird input substitute — avoid that).

---

## 3. Requirements

### Accessible name

| Method | When |
| --- | --- |
| Visible `children` text | **Preferred** |
| `aria-label` / `aria-labelledby` | When visible text is absent or insufficient |
| `loadingText` | Should keep a meaningful name while loading |

WCAG 2.5.3: if a visible label exists, the accessible name should include that text.

### Disabled and loading

| State | Requirement |
| --- | --- |
| `isDisabled` / `disabled` | Native `disabled`; not focusable/operable; announced as disabled |
| `isLoading` | Treated as disabled (`disabled={isDisabled \|\| isLoading}`); do not allow activation |
| Name while loading | Prefer `loadingText` or keep original name; spinner alone is not a name |

### Icons

- Decorative icons beside text: hide from AT (`aria-hidden`).
- Icon-only UIs: use **IconButton** (requires `aria-label`), not a bare Button with only an icon.

### Keyboard

| Key | Requirement |
| --- | --- |
| **Tab** / **Shift+Tab** | Moves focus to / from the button |
| **Enter** / **Space** | Activates |
| Disabled / loading | No activation |

No keyboard trap. Visible focus indicator (2.4.7). Target size ≥ 24×24 CSS px (2.5.8).

---

## 4. WCAG highlights

| SC | Why |
| --- | --- |
| **2.1.1 / 2.4.7** | Keyboard operability; focus visible |
| **2.5.3 / 4.1.2** | Name matches visible label; role/name determinable |
| **2.5.8** | Minimum target size |
| **1.4.3 / 1.4.11** | Text and UI contrast |

---

## 5. Checklist

- [ ] Renders `<button>` with explicit or default `type`
- [ ] Has an accessible name (text or aria-label)
- [ ] Icons decorative when accompanying text
- [ ] Disabled / loading not activatable
- [ ] Enter / Space activate; focus visible
- [ ] Contrast and target size OK
