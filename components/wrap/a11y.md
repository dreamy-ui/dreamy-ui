# Wrap — Accessibility

`Wrap` is a flex layout primitive that allows children to wrap onto new lines (`div`). It is **not** an APG interactive widget.

---

## Semantics

- Generic container; no default ARIA role.
- Wrapping is visual only — it does not create rows, lists, or groups in the accessibility tree.
- Children retain their own semantics and focus behavior.

---

## Reading order

- Follow DOM order for reading and keyboard navigation (WCAG **1.3.2**).
- Wrapped lines do not start a new “row” for assistive tech; content is still a flat sequence of children.
- If wrap reorders content visually at breakpoints, verify the spoken/tab order still matches intent.

---

## Authoring checklist

| Do | Don't |
| --- | --- |
| Wrap chips, tags, or buttons that remain independently labeled | Rely on line breaks to convey grouping or hierarchy |
| Ensure each interactive child has its own name | Use Wrap as a toolbar/list without the appropriate role when needed |
| Keep tab order contiguous and predictable | Convey meaning only by which items share a visual line |

---

## WCAG notes

| Criterion | Relevance |
| --- | --- |
| **1.3.1** Info and Relationships | Grouping needs real structure (`ul`, `role="group"`, fieldset), not wrap lines |
| **1.3.2** Meaningful Sequence | DOM order is the sequence |
| **1.3.3** Sensory Characteristics | Avoid “items on the second line” instructions |
