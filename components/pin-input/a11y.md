# Pin Input — Accessibility Requirements

Segmented one-character fields for codes (PIN, OTP). Compound: `PinInput.Root` + multiple `PinInput.Field` inputs.

**APG:** No dedicated pin pattern — treat as related text inputs; follow [APG Patterns](https://www.w3.org/WAI/ARIA/apg/patterns/) form practices and HTML OTP autocomplete guidance.

---

## Dreamy form composition

Standalone control; compose **inside `Field`**. Field owns the overall label/hint/error for the **code as a whole**. Do not add Field chrome into PinInput.

Each segment is an `Input`; the **group** must still expose a collective accessible name and descriptions. Prefer:

- Field.Label names the pin group
- Wire `aria-describedby` / `aria-invalid` onto the group or each field consistently
- Avoid only labeling each cell as “Please enter your pin code” without a Field label

```tsx
<Field>
  <Field.Label>Verification code</Field.Label>
  <PinInput.Root otp>
    <PinInput.Field />
    <PinInput.Field />
    <PinInput.Field />
    <PinInput.Field />
  </PinInput.Root>
  <Field.Error>Invalid code</Field.Error>
</Field>
```

---

## Anatomy

| Part | Component | A11y role |
| --- | --- | --- |
| **Root** | `PinInput.Root` | Group container (`group` / labelled region) |
| **Field** | `PinInput.Field` | Native text/tel input — one character |

### Connections

```
Field.Label ──► Root (aria-labelledby) and/or first field
Field.Hint+Error ──► aria-describedby on fields or group
Fields ──► sequential focus; Backspace/arrows move between cells
otp ──► autocomplete="one-time-code" on fields
```

---

## Requirements

### Roles & grouping

- Root should expose **group** semantics: `role="group"` + `aria-labelledby` (Field label), so AT understands one code, not unrelated inputs.
- Each field: native `<input>`, `maxLength={1}`, appropriate `inputMode` (`numeric` for number type).
- Masked mode may use `type="password"` — still provide a clear group name.

### Naming

- Primary name: Field.Label for the whole PIN.
- Per-field names (e.g. “Digit 1 of 4”) help when navigating cell-by-cell; a single generic label on every cell is weak if Field is missing.
- Placeholder on unfocused cells must not replace the group label.

### Keyboard & focus management

| Key | Behavior |
| --- | --- |
| **Tab** | Enter/leave the pin group (avoid forcing Tab through every cell if managed focus is used — document the chosen model and keep it consistent) |
| Printable / digit | Fill cell; advance focus when valid |
| **Backspace** | Clear; move to previous when empty |
| **ArrowLeft** / **ArrowRight** | Move between cells when `manageFocus` |
| Paste | Distribute characters across cells; focus last filled |

Disabled fields skipped. Completing all cells may fire `onComplete` — do not steal focus unexpectedly.

### OTP

When `otp` is set, use `autocomplete="one-time-code"` so password managers / OS OTP can fill the code.

## Checklist

- [ ] Field provides overall accessible name for the code
- [ ] Root grouped (`role="group"` + labelledby) or equivalent
- [ ] Each cell is a native input with maxLength 1
- [ ] Arrow/Backspace/paste focus management works
- [ ] `otp` sets `autocomplete="one-time-code"`
- [ ] Invalid/hint wired from Field (`aria-invalid` / describedby)
- [ ] Masked inputs still have a clear purpose name
- [ ] Visible focus on the active cell
- [ ] Numeric `inputMode` when type is number
