# Icon — Accessibility Requirements

Guide for implementing and testing accessibility of the Dreamy UI `Icon` component.

**Anatomy:** styled `<svg>` (`dreamy("svg", icon)`). Meaning comes from usage context.

---

## 1. Purpose and semantics

Icons are either **decorative** or **meaningful**. The same component must support both.

| Mode | When | Requirement |
| --- | --- | --- |
| **Decorative** | Redundant with adjacent text, or pure ornament | `aria-hidden="true"` (and typically `focusable="false"`) |
| **Meaningful** | Icon alone conveys meaning (status, standalone control graphic) | Accessible name via `aria-label` / `aria-labelledby` / title pattern |

Prefer visible text next to the icon when possible; then mark the SVG decorative.

---

## 2. Requirements

### Decorative (most common)

```tsx
<Icon aria-hidden="true" focusable="false" />
```

- Parent control/text owns the accessible name (e.g. button “Delete” + decorative trash icon).

### Meaningful

- Provide `aria-label` (or labelled-by) that describes the **meaning**, not “icon” / “image”.
- Do not rely on hover tooltips alone for the only name.
- If the icon is the sole content of a control, put the name on the **control** (`button`/`a`), and keep the SVG `aria-hidden`.

### Icon-only controls

- Use `IconButton` / `button` / `Link` with an accessible name; Icon itself stays decorative inside the control.
- Meet target size ≥ 24×24 CSS px on the control (WCAG 2.5.8).

### Contrast

- Meaningful UI icons must meet non-text contrast ≥ 3:1 (WCAG 1.4.11).

---

## 3. WCAG 2.2 (relevant)

| SC | Level | Why |
| --- | --- | --- |
| **1.1.1 Non-text Content** | A | Meaningful icons need text alternatives; decorative are hidden |
| **1.4.11 Non-text Contrast** | AA | Icon glyphs vs background |
| **4.1.2 Name, Role, Value** | A | Named when exposed; hidden when decorative |

---

## 4. Checklist

- [ ] Decorative icons: `aria-hidden="true"`
- [ ] Meaningful standalone icons: explicit accessible name
- [ ] Icon-only buttons/links: name on the control, SVG hidden
- [ ] No “icon” / “image of…” fluff in labels
- [ ] Contrast OK for meaningful icons
