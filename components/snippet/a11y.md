# Snippet — Accessibility Requirements

Guide for implementing and testing accessibility of the Dreamy UI `Snippet` compound component.

**Anatomy:** `Root` → optional `Header` (icon + label + copy `IconButton`) → `Body` (`pre` > `code`).

---

## 1. Purpose and semantics

Snippet displays **code / command text** and optionally a **Copy** action. The code is content; the copy control is the interactive widget.

---

## 2. Per-part requirements

| Part | Requirement |
| --- | --- |
| **Root** | Grouping container; default header `aria-label` (“Snippet”) when provided by hooks — keep or override with a clearer name |
| **Header icon** | Decorative by default (`aria-hidden` on default terminal SVG) |
| **Header content** | Visible label for the snippet (e.g. filename / shell) |
| **Copy button** | Real button with accessible name (`aria-label` “Copy” / “Copy to clipboard”); icons `aria-hidden` |
| **Body / `pre` / `code`** | Preserve code semantics; ensure text is selectable and present in the accessibility tree |

### Copy feedback

- After copy, visual check icon must not leave the button unnamed; keep a stable or updated accessible name (e.g. “Copied” if state is announced).
- Prefer a polite status update for “Copied” when feedback is only visual (WCAG 4.1.3).

### Keyboard

| Key | On copy button |
| --- | --- |
| **Tab** | Focuses the button |
| **Enter / Space** | Activates copy |

Do not trap focus inside the snippet.

## 3. WCAG 2.2 (relevant)

| SC | Level | Why |
| --- | --- | --- |
| **1.1.1 Non-text Content** | A | Copy/check icons need names via the button |
| **1.3.1 Info and Relationships** | A | `pre`/`code` structure |
| **2.1.1 Keyboard** | A | Copy operable by keyboard |
| **2.4.7 Focus Visible** | AA | Focus on copy control |
| **2.5.8 Target Size (Minimum)** | AA | Copy target ≥ 24×24 CSS px |
| **4.1.2 Name, Role, Value** | A | Button name/role |
| **4.1.3 Status Messages** | AA | Optional “Copied” status |

---

## 4. Checklist

- [x] Copy control is a named button; icons are decorative
- [x] Header decorative icon is `aria-hidden`
- [x] Code text is available to AT / selection
- [x] Keyboard activates copy; no focus trap
- [x] Copy success feedback is not color-only
