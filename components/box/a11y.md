# Box — Accessibility

`Box` is a layout primitive (`div`). It is **not** an APG interactive widget. Accessibility work is about preserving child semantics and avoiding misuse as a fake control.

---

## Semantics

- Renders a generic container. No default ARIA role.
- Do not use `Box` as a button, link, or other control without a native element (or correct role + keyboard + name).
- Prefer semantic HTML (`main`, `nav`, `section`, headings, lists) for structure; use `Box` for styling/layout only.

---

## Authoring checklist

| Do | Don't |
| --- | --- |
| Keep DOM order aligned with reading order | Rely on visual reorder alone for meaning (CSS `order`, absolute layout) |
| Pass through `role` / `aria-*` only when needed | Wrap interactive content in a way that breaks labels or focus |
| Use headings/landmarks for page structure | Put click handlers on a bare `Box` without role/keyboard |

---

## WCAG notes

| Criterion | Relevance |
| --- | --- |
| **1.3.1** Info and Relationships | Structure comes from real elements/roles, not visual boxes |
| **1.3.2** Meaningful Sequence | DOM order should match the intended reading order |
| **4.1.2** Name, Role, Value | Only if you add an interactive role — then name + keyboard are required |
