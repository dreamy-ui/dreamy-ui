# Grid — Accessibility

`Grid` and `GridItem` are CSS grid layout primitives (`div`). They are **not** the APG [Grid (Interactive) Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/grid/) and must not be confused with `role="grid"`.

---

## Semantics

- Layout-only containers; no default ARIA roles.
- CSS columns/rows/`colSpan` etc. are not exposed as a data grid to assistive technologies.
- Do **not** set `role="grid"` unless you intentionally implement a full interactive grid widget (keyboard model, focus management, cells).

---

## Reading order

- Assistive tech follows **DOM order**, not visual grid placement (WCAG **1.3.2**).
- Place items in the DOM in the order users should hear/tab through them.
- Be careful with dense responsive rearrangements that change visual order without updating DOM order.

---

## Authoring checklist

| Do | Don't |
| --- | --- |
| Use Grid for page/card layout of semantic content | Assume column position conveys structure to screen readers |
| Prefer real lists/tables when those semantics apply | Use CSS grid as a substitute for `<table>` data tables |
| Keep focusable content in a logical sequence | Add `role="grid"` for visual layout alone |

---

## WCAG notes

| Criterion | Relevance |
| --- | --- |
| **1.3.1** Info and Relationships | Use lists, headings, or tables when relationships matter |
| **1.3.2** Meaningful Sequence | DOM order must remain meaningful |
| **2.4.3** Focus Order | Tab order should not jump unpredictably across the visual grid |
