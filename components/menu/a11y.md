# Menu — Accessibility Requirements

Guide for implementing and testing accessibility of the Dreamy UI `Menu` component.

**APG patterns:** [Menu and Menubar](https://www.w3.org/WAI/ARIA/apg/patterns/menu/), [Menu Button](https://www.w3.org/WAI/ARIA/apg/patterns/menu-button/)

---

## 1. Purpose and semantics

A menu presents a list of actions or options from a **menu button**. Focus and keyboard navigation follow APG menu behavior (arrow keys among items, Escape to close, return focus to the trigger).

Dreamy `Menu` composes `Popover` for positioning/portal; a11y semantics should follow **menu**, not dialog: prefer `role="menu"` on content and `role="menuitem"` (or `menuitemcheckbox` / `menuitemradio`) on items.

---

## 2. Component anatomy

| Part | Purpose in a11y |
| --- | --- |
| **`Menu.Root`** | Menu state + descendants registry |
| **`Menu.Trigger`** | Menu button (`aria-haspopup`, `aria-expanded`, `aria-controls`) |
| **`Menu.Content`** | Popup menu container (`role="menu"`) |
| **`Menu.Item`** | Action (`role="menuitem"`) |
| **`Menu.TriggerItem`** | Nested submenu trigger + nested `Menu.Content` |

---

## 3. Requirements

### Trigger (menu button)

- Focusable button with accessible name.
- `aria-haspopup="menu"`, `aria-expanded`, `aria-controls` → menu id.
- Opens on click; APG also supports ArrowDown / ArrowUp to open and move focus into the menu.

### Content and items

- Content: `role="menu"`; labelled by the trigger when appropriate (`aria-labelledby`).
- Items: `role="menuitem"` (or checkbox/radio variants); disabled items not activatable.
- **Roving focus / virtual focus** among items (ArrowUp / ArrowDown); Home / End optional per APG.
- Enter / Space activates the focused item and typically closes the menu.
- **Escape** closes; focus returns to `Menu.Trigger`.
- Typeahead (optional APG): printable characters move focus to matching items.

### Submenus (`Menu.TriggerItem`)

- Nested content is also `role="menu"`.
- ArrowRight opens; ArrowLeft / Escape closes submenu and restores navigation in the parent menu.
- Parent item conveys submenu presence (e.g. `aria-haspopup="menu"`, `aria-expanded`).

### Keyboard (summary)

| Key | Requirement |
| --- | --- |
| **Enter** / **Space** / **ArrowDown** | Open from trigger (as applicable) |
| **ArrowUp** / **ArrowDown** | Move among items |
| **Enter** / **Space** | Activate item |
| **Escape** | Close menu / submenu |
| **ArrowRight** / **ArrowLeft** | Open / close submenu |

## 4. WCAG (relevant)

| SC | Level | Notes |
| --- | --- | --- |
| **2.1.1 Keyboard** | A | Full menu operation by keyboard |
| **2.1.2 No Keyboard Trap** | A | Escape returns to trigger |
| **2.4.3 Focus Order** | A | Logical item order |
| **2.4.7 Focus Visible** | AA | Visible item / trigger focus |
| **2.5.8 Target Size** | AA | Items ≥ 24×24 CSS px (or exception) |
| **4.1.2 Name, Role, Value** | A | menu / menuitem / expanded |

---

## 5. Checklist

- [ ] Trigger: `aria-haspopup="menu"`, `aria-expanded`, named
- [ ] Content: `role="menu"`; items: `menuitem` (or variants)
- [ ] Arrow keys move between items; Enter activates
- [ ] Escape closes; focus returns to trigger
- [ ] Submenus: ArrowRight/Left (+ Escape) behave per APG
- [ ] Disabled items skipped / not activated
- [ ] Focus visible on trigger and items
