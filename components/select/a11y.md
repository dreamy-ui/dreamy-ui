# Select — Accessibility Requirements

Custom select: trigger button opens a popup list of options, with a visually hidden native `<select>` for form integration. Align with select-only combobox / listbox popup behavior.

**APG:** [Combobox Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/combobox/) (select-only) · [Listbox Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/listbox/)

---

## Dreamy form composition

Standalone control; compose **inside `Field`**. Field owns label/hint/error. Do not add Field chrome into Select.

Wire Field → **trigger** (and keep hidden select in sync for forms): `id`, `aria-describedby`, `aria-invalid`, `aria-required`.

```tsx
<Field>
  <Field.Label>Country</Field.Label>
  <Select.Root items={items}>
    <Select.Trigger placeholder="Select…" />
    <Select.Content />
  </Select.Root>
  <Field.Error>Required</Field.Error>
</Field>
```

## Anatomy

| Part | Component | A11y role |
| --- | --- | --- |
| **Root** | `Select.Root` | Container |
| **Hidden select** | native `<select>` (visually hidden) | Form submission / progressive enhancement |
| **Trigger** | `Select.Trigger` (`button`) | Combobox / disclosure control — primary focus |
| **Content** | `Select.Content` / `VirtualContent` | `listbox` popup |
| **Item** | auto-rendered option buttons | `option` |
| **Clear** | clear button (when clearable) | Named button (`aria-label`) |
| **Indicators** | chevron / checkmark SVGs | Decorative (`aria-hidden`) |

### Connections

```
Field.Label ──► Trigger (id / labelledby)
Field.Hint+Error ──► Trigger (aria-describedby)
Trigger aria-expanded / aria-controls ──► listbox
listbox options ──► aria-selected; keyboard focus / activedescendant
Hidden <select> ──► form name/value (sync with selection)
```

## Requirements

### Roles & state

| Element | Requirement |
| --- | --- |
| Trigger | `aria-expanded`, `aria-haspopup="listbox"`, optional `aria-controls` → listbox id |
| Popup | `role="listbox"` (single or multi per `multiple`) |
| Items | `role="option"`, `aria-selected`, `aria-disabled` when disabled |
| Selection | Visible value text on trigger; name must not rely on color alone |

Hidden native select must stay synchronized for form posts; keep it from creating a **second** tab stop that confuses AT (visually hidden + excluded from tab order as appropriate).

### Naming

- Field.Label (or `aria-label` on trigger) names the control.
- Each option’s accessible name = item label (or `renderItem` text).
- Clear button: explicit accessible name (e.g. “Clear selection”).
- Placeholder is not a permanent accessible name substitute.

### Keyboard (select-only combobox)

| Key | Behavior |
| --- | --- |
| **Tab** | Focus trigger; leave control |
| **Enter** / **Space** | Open; if open, select focused option (Enter) / toggle open (Space per design) |
| **ArrowDown** / **ArrowUp** | Open if closed; move focus among options |
| **Escape** | Close; restore focus to trigger |
| **Home** / **End** | Optional: first / last option |

On close, return focus to the trigger. No keyboard trap. Disabled items skipped.

### Multiple

When `multiple` is enabled, listbox is multi-select (`aria-multiselectable` as applicable); announce selected state per option; trigger displays selected labels clearly.

## Checklist

- [ ] Trigger has accessible name from Field / aria-label
- [ ] `aria-expanded` / `aria-haspopup` reflect open state
- [ ] Popup is a listbox; items are options with `aria-selected`
- [ ] Arrow/Enter/Escape/Space match APG select-only behavior
- [ ] Focus returns to trigger on close
- [ ] Field wires describedby / invalid / required on trigger
- [ ] Hidden select syncs value without duplicate confusing tab stop
- [ ] Clear button is named; icons decorative
- [ ] Virtualized lists keep focused option reachable/scrollable
