# Radio — Accessibility Requirements

Single-select option within a mutually exclusive set. Dreamy uses a visually hidden `input[type="radio"]` plus a decorative control.

**APG:** [Radio Group Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/radio/)

---

## Dreamy form composition

Standalone control; compose **inside `Field`** (often one Field wrapping `RadioGroup`). Field owns label/hint/error. Do not add Field chrome into Radio.

Optional `children` = inline option name. Preferred group usage:

```tsx
<Field>
  <Field.Label>Plan</Field.Label>
  <RadioGroup>
    <Radio value="pro">Pro</Radio>
    <Radio value="team">Team</Radio>
  </RadioGroup>
  <Field.Error>Select a plan</Field.Error>
</Field>
```

Each radio must expose name/role/value and accept Field wiring (`aria-describedby`, `aria-invalid`, etc.).

## Anatomy

| Part | Element | A11y role |
| --- | --- | --- |
| **Root** | styled `div` | Click association / visual container |
| **Input** | visually hidden `input[type="radio"]` | **The radio** — focus, keyboard, form value |
| **Wrapper / Control** | decorative spans | `aria-hidden="true"` |
| **Inline label** | optional `span` (`children`) | Option name |

---

## Requirements

### Role & state

- Native `input[type="radio"]` — no custom `role="radio"` required.
- Checked via native `checked`.
- All radios in a set share the same `name` (from props / group).

### RadioGroup

`RadioGroup` provides shared value and feedback context (`isDisabled`, `isInvalid`, `isRequired`, `isReadOnly`).

**Grouping roles (required for AT):**

- Prefer `role="radiogroup"` + `aria-labelledby` pointing at the group label, **or**
- `<fieldset>` + `<legend>` (e.g. via Fieldset).

Without group semantics, AT may not announce the set as a radio group. Each option still has its own accessible name.

### Naming

| Method | When |
| --- | --- |
| Group label (`Field.Label` / legend) | Names the **set** |
| `children` / `aria-label` | Names each **option** |

### Keyboard (APG)

| Key | Behavior |
| --- | --- |
| **Tab** | Moves focus into / out of the group (to the checked radio, or first if none) |
| **Arrow keys** | Move selection/focus between radios in the group |
| **Space** | Checks the focused radio if unchecked |

Roving tabindex / native radio behavior should match APG expectations. Disabled options skipped.

### Other attributes

| Attribute | Requirement |
| --- | --- |
| `id` | Stable; Field association when used |
| `aria-describedby` | Field hint/error |
| `aria-invalid` | From Field / group |
| `disabled` / `required` | From Field / group / props |
| `name` / `value` | Shared name + unique values |

- **2.5.8** Target Size (Minimum)
