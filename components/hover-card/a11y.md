# Hover Card — Accessibility Requirements

Guide for implementing and testing accessibility of the Dreamy UI `HoverCard` component.

**Related APG:** [Disclosure](https://www.w3.org/WAI/ARIA/apg/patterns/disclosure/), [Tooltip](https://www.w3.org/WAI/ARIA/apg/patterns/tooltip/), [Dialog](https://www.w3.org/WAI/ARIA/apg/patterns/dialog-modal/) (non-modal)

---

## 1. Purpose and semantics

A hover card shows **rich preview content** on pointer hover (and should remain usable from keyboard). It builds on Popover with `trigger: "hover"`.

**Role choice:**

- **Informational only** (no interactive children): `role="tooltip"` + trigger `aria-describedby` is acceptable.
- **Interactive content** (links, buttons): do **not** use `tooltip`. Prefer `role="dialog"` (non-modal) or a disclosure pattern so users can Tab into the card. Tooltips must not contain interactive elements (APG Tooltip).

Default hover-card content must remain dismissible and must not obscure critical UI without an Escape path (WCAG 1.4.13).

| **`HoverCard.Root`** | Open/close delays, portal, open state |
| **`HoverCard.Trigger`** | Hover/focus target; associates with content |
| **`HoverCard.Content`** | Floating preview panel |
| **`HoverCard.Header`** | Title / name |
| **`HoverCard.Body`** | Main preview text/media |
| **`HoverCard.Footer`** | Optional actions (implies interactive → not tooltip) |
| **`HoverCard.Arrow`** | Decorative |

## 3. Requirements

### Trigger and content

- Trigger is focusable (or wrapped so keyboard users can open it).
- Open on hover **and** keyboard focus; close on leave/blur after delay, and on **Escape**.
- Pointer can move onto Content without it closing immediately (1.4.13 “hoverable”).
- If Content is a dialog: name via Header / `aria-label`; trigger may use `aria-expanded` / `aria-controls`.
- If Content is a tooltip: trigger `aria-describedby` → content id; content `role="tooltip"`.
- No `aria-modal`; no page-level focus trap.
- Decorative arrow: hide from AT if needed.

### Keyboard / pointer

| Input | Requirement |
| --- | --- |
| **Focus** on trigger | Opens (after delay if any) |
| **Escape** | Dismisses |
| **Pointer leave** | Closes after `closeDelay`, unless pointer moved into Content |
| **Tab** | If interactive children exist, they must be reachable |

---

## 5. Checklist

- [x] Opens on keyboard focus as well as hover
- [x] Escape dismisses; pointer can move onto content
- [x] Role matches content (tooltip vs dialog) — no interactive tooltip
- [x] Trigger association (`aria-describedby` or expanded/controls) is correct
- [x] Not modal; no accidental focus trap
- [x] Reduced motion respected for transitions
