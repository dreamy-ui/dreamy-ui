# Skeleton — Accessibility Requirements

Guide for implementing and testing accessibility of the Dreamy UI `Skeleton` / `SkeletonText` components.

**Anatomy:** placeholder `div`(s). `SkeletonText` renders multiple line placeholders in a wrapper.

---

## 1. Purpose and semantics

Skeleton is a **visual loading placeholder**. It is not interactive and usually conveys no unique information beyond “content is loading.”

| Mode | When | Requirement |
| --- | --- | --- |
| **Decorative** | Parent/region already announces loading | `aria-hidden="true"` on skeleton(s) |
| **Status** | Skeleton is the only loading cue | Region/status name + optional `aria-busy` on the container that will receive content |

Prefer announcing loading **once** at the region level — not once per skeleton bar.

---

## 2. Requirements

- Do **not** make Skeleton focusable or a tab stop.
- Do **not** block keyboard navigation of the rest of the page while skeletons paint (WCAG 2.1.2). Disable only the controls that are truly unavailable.
- When content arrives, remove skeletons and ensure the loaded content is in the accessibility tree (replace, don’t leave hidden empty placeholders).
- If a parent sets `aria-busy="true"` while loading, clear it when content is ready.
- Pulse/shimmer animations must respect `prefers-reduced-motion`.

### SkeletonText

- Treat line placeholders as decorative; hide the wrapper or lines from AT when status is handled elsewhere.
- Do not invent fake text content for AT inside each line.

---

## 3. WCAG 2.2 (relevant)

| SC | Level | Why |
| --- | --- | --- |
| **1.1.1 Non-text Content** | A | If skeleton alone signals loading, provide a text alternative / status |
| **2.1.2 No Keyboard Trap** | A | Loading placeholders must not trap focus |
| **2.2.2 Pause, Stop, Hide** | A | Decorative motion should not be required to understand the UI |
| **4.1.3 Status Messages** | AA | Loading state available without moving focus |

---

## 4. Checklist

- [x] Skeletons are not focusable
- [x] Decorative skeletons are `aria-hidden` when status is announced elsewhere
- [x] Loading announced at most once (region/status), not per bar
- [x] `aria-busy` cleared when content loads
- [x] Reduced motion respected
