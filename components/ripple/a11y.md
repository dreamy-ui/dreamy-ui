# Ripple — Accessibility Requirements

Guide for implementing and testing accessibility of the Dreamy UI `Ripple` component (used with `useRipple`).

**Not a primary widget.** Ripple is a **decorative** ink effect on pointer interaction. The host control (button, list item, etc.) owns name, role, keyboard, and focus.

---

## 1. Purpose and semantics

- Visual feedback only — must **not** appear in the accessibility tree as interactive content.
- Must **not** steal focus, receive pointer events that block the host, or announce status.
- Prefer [APG Button](https://www.w3.org/WAI/ARIA/apg/patterns/button/) (or the host’s pattern) for the interactive element that triggers ripples.

---

## 2. Decorative markup (required)

| Requirement | Detail |
| --- | --- |
| Hide from AT | Each ripple node: `aria-hidden="true"` (and stay non-focusable) |
| No names/roles | Do not set `role`, `aria-label`, or live regions on ripple nodes |
| Pointer | Keep `pointer-events: none` so the host receives all interaction |
| Disable path | Honor global `disableRipple` / `isDisabled` from `useRipple` without changing host a11y |

---

## 3. Reduced motion (required)

Ripple is motion-from-interaction. When `prefers-reduced-motion: reduce` (or Dreamy `reduceMotion` / disable-ripple) is active:

- Do **not** create or animate ripples, **or**
- Skip animation and clear immediately.

Mapped to [WCAG 2.3.3 Animation from Interactions](https://www.w3.org/WAI/WCAG22/Understanding/animation-from-interactions.html) (AAA).

---

## 4. Host control (not Ripple’s job)

The element that calls `onPointerDown` / `onClick` from `useRipple` must still meet:

- Keyboard operability ([2.1.1](https://www.w3.org/WAI/WCAG22/Understanding/keyboard.html)) — ripples are pointer-only; keyboard activation must work without them
- Visible focus ([2.4.7](https://www.w3.org/WAI/WCAG22/Understanding/focus-visible.html)) independent of the ripple
- Name, role, value ([4.1.2](https://www.w3.org/WAI/WCAG22/Understanding/name-role-value.html))

---

## 5. WCAG 2.2 (relevant)

| SC | Level | Why |
| --- | --- | --- |
| **1.1.1 Non-text Content** | A | Decorative → empty/hidden alternative (`aria-hidden`) |
| **2.3.3 Animation from Interactions** | AAA | Respect reduced motion / disable ripple |
| **2.1.1 Keyboard** | A | Host must work without relying on ripple |

Contrast of the ink vs host is optional decorative chrome; do not rely on ripple for meaning ([1.4.1 Use of Color](https://www.w3.org/WAI/WCAG22/Understanding/use-of-color.html)).

---

## 6. Checklist

- [x] Ripple nodes are `aria-hidden="true"` and non-focusable
- [x] `pointer-events: none` on ripple surfaces
- [x] No `aria-live` / status announcements from ripple
- [x] Reduced motion or disable-ripple suppresses the effect
- [x] Host control remains fully keyboard-operable and named
