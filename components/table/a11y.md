# Table — Accessibility Requirements

Guide for implementing and testing accessibility of the Dreamy UI `Table` component.

**APG:** [Table Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/table/) — prefer native HTML table elements (Dreamy’s approach).

---

## 1. Purpose and semantics

A table presents **static tabular data**. Cells are not a single composite widget; interactive controls inside cells are separate tab stops. For spreadsheet-like interaction, use a **grid** pattern instead.

---

## 2. Component anatomy

| Part | Element (Dreamy) | Role in a11y tree | Purpose |
| --- | --- | --- | --- |
| **Root** | `<div>` | Presentation / layout wrapper | Scroll/style container |
| **Table** | `<table>` | Table | Tabular structure |
| **Header** | `<thead>` | Row group | Column headers |
| **Body** | `<tbody>` | Row group | Data rows |
| **Row** | `<tr>` | Row | One record / header row |
| **ColumnHeader** | `<th>` | Column header | Column title |
| **Cell** | `<td>` | Cell | Data |
| **Caption** | `<caption>` | Caption | Accessible name / description |

```
div (Root) → table → caption? → thead/tr/th → tbody/tr/td
```

---

## 3. Requirements

### Prefer native semantics

- Use `<table>`, `<thead>`, `<tbody>`, `<tr>`, `<th>`, `<td>`, `<caption>` — do not rebuild with `div` + ARIA unless necessary (APG strongly prefers HTML).
- Column headers: `<th scope="col">` (or implicit in `<thead>`).
- Row headers when needed: `<th scope="row">`.

### Name and description

- Prefer `<Caption>` for the table’s accessible name.
- Alternatively `aria-label` / `aria-labelledby` on `<table>`.
- Longer descriptions: `aria-describedby` to explanatory text.

### Relationships

- Do not use tables for pure layout.
- Mark sortable headers with `aria-sort` when sorting is implemented ([APG sortable table](https://www.w3.org/WAI/ARIA/apg/patterns/table/examples/sortable-table/)).
- For virtualized / partial DOM: `aria-rowcount`, `aria-colcount`, `aria-rowindex`, `aria-colindex` as needed.

### Keyboard

- Static tables: no table-specific keys; interactive cells participate in normal Tab order.
- Many focusable cells → consider grid pattern to shorten tab sequence (APG note).

---

## 5. Checklist

- [ ] Real `<table>` markup (not layout-only divs)
- [ ] Column (and row, if needed) headers via `<th>`
- [ ] Caption or `aria-label` / `aria-labelledby` names the table
- [ ] Sortable columns expose `aria-sort` when applicable
- [ ] Interactive controls inside cells are keyboard operable
- [ ] Not used for non-tabular layout
